  <action>
The PubSubRegistry itself does NOT need to change -- it already works as a standalone struct. What changes:

1. **Shard owns PubSubRegistry**: Add `pub pubsub_registry: PubSubRegistry` field to the `Shard` struct in `src/shard/mod.rs`. Initialize with `PubSubRegistry::new()` in `Shard::new()`.

2. **PUBLISH fan-out in connection handler**: When a PUBLISH command arrives:
   a. Publish to the local shard's PubSubRegistry (local subscribers)
   b. Fan out to all other shards: for each other shard, push `ShardMessage::PubSubFanOut { channel, message }` to the SPSC producer for that shard.
   c. The count returned = local_count + remote_count. For simplicity, return local count immediately (remote delivery is best-effort, same as Redis -- PUBLISH return value is approximate).

   In the sharded connection handler:
   ```rust
   if cmd.eq_ignore_ascii_case(b"PUBLISH") {
       // Publish locally
       let local_count = shard.pubsub_registry.publish(&channel, &message);
       // Fan out to all other shards
       for target_shard in 0..num_shards {
           if target_shard == my_shard { continue; }
           let target_idx = ChannelMesh::target_index(my_shard, target_shard);
           let msg = ShardMessage::PubSubFanOut {
               channel: channel.clone(),
               message: message.clone(),
           };
           let _ = dispatch_tx[target_idx].try_push(msg); // best-effort
       }
       responses.push(Frame::Integer(local_count));
   }
   ```

3. **Shard event loop handles PubSubFanOut**: In the SPSC consumer drain loop:
   ```rust
   ShardMessage::PubSubFanOut { channel, message } => {
       self.pubsub_registry.publish(&channel, &message);
   }
   ```

4. **SUBSCRIBE/UNSUBSCRIBE**: These operate on the local shard's PubSubRegistry only. Since the subscriber's connection lives on this shard, its PubSubRegistry entry is local. No cross-shard needed for subscribe/unsubscribe.

5. **Connection cleanup on disconnect**: `pubsub_registry.unsubscribe_all(subscriber_id)` and `pubsub_registry.punsubscribe_all(subscriber_id)` -- already works since it's per-shard and the subscriber is local.

6. **Remove global PubSubRegistry from listener.rs**: The old code creates `Arc::new(Mutex::new(PubSubRegistry::new()))` and passes it to every connection. In the sharded path, this is no longer needed -- each shard owns its own.
