```rust
pub enum FailoverState {
    None,
    WaitingDelay { start_ms: u64, delay_ms: u64 },
    WaitingVotes { epoch: u64, votes_received: u32, votes_needed: u32 },
}
```

3. Add to ClusterState:
   - `pub failover_state: FailoverState` — initialized to FailoverState::None in ClusterState::new()

4. Add helper method to ClusterState:
```rust
pub fn quorum(&self) -> u32 {
    let masters = self.master_count() as u32;
    masters / 2 + 1
}
```

**In src/cluster/gossip.rs — merge_gossip_into_state():**

After updating the sender node, process gossip sections for PFAIL reports. For each gossip section where `flags == 2` (Pfail) or `flags == 3` (Fail):
- Extract the node_id from the section
- Extract the sender's node_id (the reporter)
- In the target node's pfail_reports, insert reporter -> now_ms()
- Then call `crate::cluster::failover::try_mark_fail_with_consensus(state, &target_node_id)` to check if majority is reached

**In src/cluster/failover.rs:**

1. Add `use std::collections::HashMap;` and `use std::time::{SystemTime, UNIX_EPOCH};`

2. Add `try_mark_fail_with_consensus(state: &mut ClusterState, node_id: &str) -> bool`:
   - Get the node's pfail_reports count
   - Quorum = state.quorum()
   - If pfail_reports.len() as u32 >= quorum AND node is currently Pfail, transition to Fail
   - Return true if transitioned
   - Clean up stale pfail_reports older than 2 * node_timeout_ms (use 30_000ms as default for now)

3. Add `compute_failover_delay(replica_rank: u32) -> u64`:
   - Returns `500 + rand::thread_rng().gen_range(0..500) + (replica_rank as u64 * 1000)`
   - This implements the Redis spec jittered delay with replica ranking

4. Add `run_election_task(cluster_state: Arc<RwLock<ClusterState>>, self_addr: SocketAddr, my_repl_offset: u64, vote_rx: tokio::sync::mpsc::Receiver<String>)`:
   - This is an async function that:
     a. Computes replica_rank (0 for now, ranking by offset will be used when multiple replicas exist)
     b. Sleeps for compute_failover_delay(replica_rank) ms
     c. Locks state, increments epoch, builds a FailoverAuthRequest GossipMessage (msg_type = FailoverAuthRequest, config_epoch = new epoch)
     d. Sends FailoverAuthRequest to ALL known masters via TCP (connect to bus_port, send length-prefixed serialized message)
     e. Sets state.failover_state = WaitingVotes { epoch, votes_received: 1 (self-vote), votes_needed: quorum }
     f. Loops on vote_rx with 5-second timeout: increment votes_received for each ack
     g. If votes_received >= votes_needed: call check_and_initiate_failover(), set failover_state = None
     h. On timeout: reset failover_state = None, log warning

5. Update existing check_and_initiate_failover to also persist nodes.conf after promotion (call state.save_nodes_conf or just note it for the caller).

6. Add tests:
   - test_try_mark_fail_needs_majority: 5 masters, need 3 pfail_reports to mark FAIL
   - test_compute_failover_delay_includes_rank: rank 0 delay < rank 1 delay (statistical)
