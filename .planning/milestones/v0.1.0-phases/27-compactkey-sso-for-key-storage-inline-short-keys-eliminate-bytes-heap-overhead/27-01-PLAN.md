Create `src/storage/compact_key.rs` with a 24-byte CompactKey struct following the CompactValue SSO pattern from Phase 9.

Layout (24 bytes total, same size as Bytes):
- Inline path: `data: [u8; 24]` where `data[0]` is length (0..=23, high bit clear), `data[1..1+len]` is key data
- Heap path: `data[0]` has high bit set (0x80 | len_hi), `data[1]` = len_mid, `data[2..4]` = len_lo (little-endian u16), `data[4..12]` = raw pointer to heap allocation (Box<[u8]>), `data[12..24]` = first 12 bytes of key as inline prefix for fast comparison rejection

Implement these traits:
- `AsRef<[u8]>` — returns inline slice or heap slice
- `Borrow<[u8]>` — delegates to AsRef (CRITICAL: DashTable segment uses `K: Borrow<Q>, Q: Eq` for lookups with `&[u8]` keys)
- `Hash` — hash the byte content via AsRef
- `Eq + PartialEq` — compare byte content via AsRef
- `Clone` — inline: copy bytes; heap: allocate new Box<[u8]> and copy
- `Drop` — inline: no-op; heap: reconstruct Box<[u8]> from raw pointer and drop
- `Debug` — show as UTF-8 string if valid, hex otherwise
- `From<Bytes>` — convert from Bytes (used in db.rs set paths)
- `From<&[u8]>` — convert from byte slice (replaces Bytes::copy_from_slice(key) pattern)
- `From<&str>` — convenience for tests
- `PartialEq<[u8]>` — compare with byte slices directly
- `as_bytes(&self) -> &[u8]` — public method returning the key bytes

Size assertion: `const _: () = assert!(std::mem::size_of::<CompactKey>() == 24);`

The inline threshold is 23 bytes (data[0] is the length byte, leaving 23 bytes for data).
The heap threshold check: if key.len() > 23, use heap path.

Heap path unsafe discipline (follow CompactValue pattern):
- `Box::into_raw(Box::from(slice))` to get raw pointer on creation
- Store pointer as `usize` in `data[4..12]` (little-endian)
- `Box::from_raw(std::slice::from_raw_parts_mut(ptr, len))` in Drop and Clone
- Prefix bytes `data[12..24]` filled from key[0..min(12, len)] for fast mismatch rejection

Add `pub mod compact_key;` to `src/storage/mod.rs`.

Include unit tests in the same file:
- Inline key roundtrip (empty, 1 byte, 23 bytes)
- Heap key roundtrip (24 bytes, 100 bytes)
- Hash consistency (same bytes = same hash)
- Eq consistency (same bytes = equal, different = not equal)
- Clone correctness for both paths
- Drop safety (no double free)
- From<Bytes> conversion
- From<&[u8]> conversion
- Borrow<[u8]> returns correct slice
- Size is exactly 24 bytes
