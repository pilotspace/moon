The principle: wherever code receives keys FROM DashTable iteration (type changes from &Bytes to &CompactKey), update. Wherever code passes keys TO Database.set(), convert to CompactKey. Wire-level key extraction from Frame remains Bytes — convert at the Database boundary.

**src/persistence/snapshot.rs:**
- OverflowEntry key field: if typed as `Bytes`, change to `CompactKey` or keep as `Bytes` and convert at insertion
- Snapshot iteration over DashTable segments: key type changes from `&Bytes` to `&CompactKey`
- Serialization: use `key.as_ref()` to get `&[u8]` for writing bytes to disk (format unchanged)
- Deserialization/restore: create `CompactKey::from(bytes_slice)` when inserting restored keys

**src/persistence/rdb.rs:**
- `save_entry` key param: if `&Bytes`, change to `&[u8]` or `&CompactKey` — use `.as_ref()` for serialization
- RDB restore: keys loaded as byte slices, convert to CompactKey via `CompactKey::from(slice)` when inserting into Database

**src/persistence/aof.rs:**
- AOF replay inserts keys into Database — convert Bytes from parsed RESP frames to CompactKey via `CompactKey::from(bytes)`
- AOF serialization reads keys from DashTable — use `.as_ref()` for wire format

**src/shard/dispatch.rs:**
- `MultiExecute` commands field `Vec<(Bytes, Frame)>` — the Bytes here is the routing key extracted from Frame, NOT a CompactKey. This stays as Bytes for routing. The conversion to CompactKey happens inside Database.set() which is called by command handlers, not dispatch.
- If dispatch directly calls db.set(), update key to CompactKey::from(bytes)

**src/cluster/migration.rs:**
- Slot migration iterates DashTable keys — update from `&Bytes` to `&CompactKey`
- Key serialization for migration protocol: use `.as_ref()` to get `&[u8]`

**src/blocking/mod.rs:**
- BlockingRegistry maps waiting keys — if using `Bytes` as key type internally for wait queues, these are wire-level keys from commands, not DashTable keys. Likely stays as Bytes.
- If it touches DashTable key iteration, update accordingly

**src/tracking/invalidation.rs:**
- Tracking invalidation uses key bytes for notification — use `.as_ref()` on CompactKey

After all changes, run full test suite to catch any remaining type mismatches. Fix compilation errors by following the pattern: DashTable keys = CompactKey, wire/Frame keys = Bytes, conversion at Database boundary.
