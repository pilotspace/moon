Insert a software prefetch hint after computing the segment storage index in `get()`, `get_mut()`, `contains_key()`, `insert()`, and `remove()` methods. The prefetch overlaps memory fetch latency (~10ns on L2/L3 miss) with the subsequent h2/home_buckets hash computation (~6ns).

For each lookup method, add the prefetch between `let seg_idx = self.directory[dir_idx];` and `let h2_val = h2(hash);`:

```rust
// Prefetch segment while computing home bucket
let seg_ptr = self.segments[seg_idx].as_ref() as *const _ as *const i8;
#[cfg(target_arch = "x86_64")]
unsafe { core::arch::x86_64::_mm_prefetch(seg_ptr, core::arch::x86_64::_MM_HINT_T0); }
#[cfg(target_arch = "aarch64")]
unsafe { core::arch::aarch64::__prefetch(seg_ptr as *const i8, 0, 3); }
```

Notes:
- Use `as_ref()` on `Box<Segment>` to get `&Segment`, then cast to raw pointer. Do NOT cast `&self.segments[seg_idx]` (that's the Box pointer, not the segment data).
- `_MM_HINT_T0` fetches into L1 (closest cache). For aarch64, locality=3 means same.
- Prefetch is a hint; it is safe even if the pointer is subsequently not accessed.
- The `unsafe` block is sound because `seg_ptr` is derived from a valid reference.
- Apply to all 5 methods: `get`, `get_mut`, `contains_key`, `insert`, `remove` (and `remove_entry` if it exists).

  <action>
Replace the current `PartialEq` implementation that calls `self.as_ref() == other.as_ref()` (which branches on inline/heap, calls as_bytes(), then does memcmp) with a fast path for inline keys that compares the raw 24-byte `data` array directly.

Replace the existing `impl PartialEq for CompactKey` (lines 127-132) with:

```rust
impl PartialEq for CompactKey {
    #[inline(always)]
    fn eq(&self, other: &Self) -> bool {
        // Fast path: if both are inline (high bit clear), compare raw 24-byte arrays.
        // This compiles to 3 QWORD comparisons with no function calls.
        if (self.data[0] | other.data[0]) & HEAP_FLAG == 0 {
            self.data == other.data
        } else {
            // Slow path: at least one is heap-allocated, compare byte slices.
            self.as_bytes() == other.as_bytes()
        }
    }
}
```

Key points:
- `(self.data[0] | other.data[0]) & HEAP_FLAG == 0` checks both are inline in one branch.
- `self.data == other.data` for inline keys is correct because: data[0] is the length byte, and unused trailing bytes are always 0 (set during `From<&[u8]>` which starts with `[0u8; 24]`).
- `#[inline(always)]` ensures this is inlined at call sites (it was `#[inline]` before, change to `#[inline(always)]`).
- Do NOT change the `PartialEq<[u8]>` impl (line 150-155) -- that one must still use `as_ref()`.
