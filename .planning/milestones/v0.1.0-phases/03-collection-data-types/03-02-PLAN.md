Create `src/command/hash.rs` with the following command handlers. Each follows the pattern `pub fn cmd(db: &mut Database, args: &[Frame]) -> Frame`. Include local helpers (err_wrong_args, extract_bytes) or use `pub(crate)` from string.rs.

**Commands to implement:**

1. `hset(db, args)` -- HSET key field value [field value ...]. Args: key + N field-value pairs (args.len() >= 3, odd count). Use `db.get_or_create_hash(key)`. For each pair, insert into HashMap. Count only NEW fields (where `insert()` returns None). Return `Frame::Integer(count_of_new_fields)`. Per Pitfall 4: count inserts that return None, not total.

2. `hget(db, args)` -- HGET key field. Args: key + field (2). Use `db.get_hash(key)`. Return field value as BulkString or Null if field/key missing.

3. `hdel(db, args)` -- HDEL key field [field ...]. Args: key + 1+ fields. Use `db.get_or_create_hash(key)` (or get_mut pattern). Remove each field, count removals. Return Integer(count). If hash becomes empty after deletions, remove the key entirely.

4. `hmset(db, args)` -- HMSET key field value [field value ...]. Same as HSET but always returns OK (legacy command). Args: key + N field-value pairs (args.len() >= 3, odd count).

5. `hmget(db, args)` -- HMGET key field [field ...]. Args: key + 1+ fields. Return Array of values (Null for missing fields). Use `db.get_hash(key)`.

6. `hgetall(db, args)` -- HGETALL key. Args: key only (1). Return Array of alternating field, value pairs. Empty array if key missing.

7. `hexists(db, args)` -- HEXISTS key field. Args: key + field (2). Return Integer(1) if field exists, Integer(0) if not (or key missing).

8. `hlen(db, args)` -- HLEN key. Args: key (1). Return Integer(hash.len()) or Integer(0) if key missing.

9. `hkeys(db, args)` -- HKEYS key. Args: key (1). Return Array of all field names. Empty array if key missing.

10. `hvals(db, args)` -- HVALS key. Args: key (1). Return Array of all values. Empty array if key missing.

11. `hincrby(db, args)` -- HINCRBY key field increment. Args: key + field + increment (3). Parse increment as i64. Get or create hash, get field value (default "0"), parse as i64, add increment, store back as string bytes. Return Integer(new_value). Error if field value is not an integer.

12. `hincrbyfloat(db, args)` -- HINCRBYFLOAT key field increment. Like HINCRBY but with f64. Return BulkString of new value (formatted without trailing zeros but keeping at least one decimal if needed -- use Redis format: if value is integer-like, format as integer string).

13. `hsetnx(db, args)` -- HSETNX key field value. Args: key + field + value (3). Set field only if it does not exist. Return Integer(1) if set, Integer(0) if field already exists.

14. `hscan(db, args)` -- HSCAN key cursor [MATCH pattern] [COUNT count]. Args: key + cursor + optional MATCH/COUNT. Collect hash fields into a Vec, iterate from cursor position up to COUNT items. Return Array of [next_cursor_string, Array of alternating field, value]. Use glob_match for MATCH filtering. Return cursor "0" when done.

**Tests (inline `#[cfg(test)] mod tests`):**
- hset: create hash, set fields, verify new-field count vs update count
- hget: get existing field, get missing field, get from missing key
- hdel: delete fields, verify count, verify hash removal when empty
- hmset/hmget: batch set then batch get, verify nulls for missing fields
- hgetall: verify alternating field/value array
- hexists/hlen: check existence and size
- hkeys/hvals: verify field names and values returned
- hincrby: increment field, verify result, error on non-integer field
- hincrbyfloat: increment with float, verify string result
- hsetnx: set only if not exists, verify no overwrite
- hscan: iterate with cursor, match pattern filter
- WRONGTYPE: call hset on a key holding a string -> WRONGTYPE error
