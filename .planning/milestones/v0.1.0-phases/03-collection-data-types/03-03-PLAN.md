**Commands to implement:**

1. `lpush(db, args)` -- LPUSH key element [element ...]. Args: key + 1+ elements. Use `db.get_or_create_list(key)`. For each element left-to-right, call `push_front`. Per Pitfall 5: `LPUSH mylist a b c` results in `[c, b, a]` because each is pushed to the front in order. Return Integer(list.len()).

2. `rpush(db, args)` -- RPUSH key element [element ...]. Same but `push_back`. Return Integer(list.len()).

3. `lpop(db, args)` -- LPOP key [count]. Without count: pop_front, return BulkString or Null. With count: pop count elements from front, return Array. If list becomes empty, remove the key.

4. `rpop(db, args)` -- RPOP key [count]. Same but pop_back.

5. `llen(db, args)` -- LLEN key. Return Integer(list.len()) or 0 if key missing.

6. `lrange(db, args)` -- LRANGE key start stop. Parse start/stop as i64, convert to usize using Redis index semantics: negative indices count from end (-1 = last). Clamp to list bounds. Return Array of elements in range [start, stop] inclusive.

7. `lindex(db, args)` -- LINDEX key index. Parse index as i64, negative counts from end. Return BulkString at position or Null if out of range.

8. `lset(db, args)` -- LSET key index element. Parse index as i64, negative counts from end. Set element at position. Return OK. Error if index out of range: "ERR index out of range".

9. `linsert(db, args)` -- LINSERT key BEFORE|AFTER pivot element. Find pivot in list (linear scan), insert element before or after it. Return Integer(new_list_len) or Integer(-1) if pivot not found. If key missing, return Integer(0).

10. `lrem(db, args)` -- LREM key count element. Remove `count` occurrences of element. count > 0: remove from head; count < 0: remove from tail; count == 0: remove all. Return Integer(removed_count). If list becomes empty, remove the key.

11. `ltrim(db, args)` -- LTRIM key start stop. Keep only elements in [start, stop] range (Redis index semantics with negatives). Drain elements outside range. Return OK. If range is empty, remove the key.

12. `lpos(db, args)` -- LPOS key element [RANK rank] [COUNT count] [MAXLEN maxlen]. Find position(s) of element. RANK N means skip first N-1 matches (1-based). COUNT 0 means return all. MAXLEN limits scan length. Without COUNT, return Integer(position) or Null. With COUNT, return Array of positions.

**Tests (inline `#[cfg(test)] mod tests`):**
- lpush: push a b c, verify lrange returns [c, b, a] (Pitfall 5)
- rpush: push a b c, verify lrange returns [a, b, c]
- lpop/rpop: pop single, pop with count, pop from empty
- llen: verify length after pushes
- lrange: positive indices, negative indices (-1 = last), out of range clamping
- lindex: positive, negative, out of range returns Null
- lset: set valid index, error on out of range
- linsert: before and after pivot, pivot not found returns -1
- lrem: remove from head (count > 0), from tail (count < 0), all (count == 0)
- ltrim: trim to subrange, trim to empty removes key
- lpos: find position, with RANK, with COUNT
- WRONGTYPE: lpush on string key returns WRONGTYPE
