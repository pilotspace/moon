Actually, the lazy-expiry `get` already handles this correctly: if expired, it removes and returns None (which means -2 for TTL). For non-expired keys, it returns the entry with its expires_at. So the existing `get` method works. But we need a way to check if a key exists WITHOUT mutating (for TYPE, TTL). Add a method:

2. **src/command/key.rs** -- Create key management command handlers:

   Create helper `fn err_wrong_args(cmd: &str) -> Frame`.

   **DEL** (`pub fn del(db: &mut Database, args: &[Frame]) -> Frame`):
   - Arity: at least 1. Return err_wrong_args("DEL") if 0.
   - For each key arg: call `db.remove(key)`. Count how many returned Some.
   - Return `Frame::Integer(count)`.

   **EXISTS** (`pub fn exists(db: &mut Database, args: &[Frame]) -> Frame`):
   - Arity: at least 1.
   - For each key arg: call `db.exists(key)`. Count how many returned true.
   - Note: Redis EXISTS counts duplicates. `EXISTS key key` returns 2 if key exists.
   - Return `Frame::Integer(count)`.

   **EXPIRE** (`pub fn expire(db: &mut Database, args: &[Frame]) -> Frame`):
   - Arity: exactly 2 (key, seconds).
   - Parse seconds as i64. If <= 0: remove key (Redis behavior: EXPIRE with 0 or negative deletes the key). Actually, Redis returns error for negative but deletes for 0. Let's match Redis: if seconds < 0, return `ERR invalid expire time in 'EXPIRE' command`. If seconds == 0, delete the key and return 1.
   - Actually checking Redis docs: EXPIRE key 0 has the effect of deleting the key. And negative values: "Since Redis 2.6, expire returns 0 and does not modify the timeout for key with a negative value." Actually in modern Redis (7+), negative expire time returns an error. Let's follow modern Redis: if seconds <= 0, return `ERR invalid expire time in 'EXPIRE' command`.
   - If key exists (lazy check): compute `Instant::now() + Duration::from_secs(seconds)`, call `db.set_expiry(key, Some(instant))`. Return `Frame::Integer(1)`.
   - If key does not exist: return `Frame::Integer(0)`.

   **PEXPIRE** (`pub fn pexpire(db: &mut Database, args: &[Frame]) -> Frame`):
   - Same as EXPIRE but uses milliseconds. `Duration::from_millis(ms)`.
   - Negative/zero: `ERR invalid expire time in 'PEXPIRE' command`.

   **TTL** (`pub fn ttl(db: &mut Database, args: &[Frame]) -> Frame`):
   - Arity: exactly 1.
   - If key does not exist (after lazy expiry): return `Frame::Integer(-2)`.
   - If key exists but no expiry (expires_at is None): return `Frame::Integer(-1)`.
   - If key exists with expiry: compute remaining = expires_at - Instant::now(). Return `Frame::Integer(remaining.as_secs() as i64)`. If remaining is zero or negative (edge case), return `Frame::Integer(0)` or delete key and return -2.
   - IMPORTANT (Pitfall 6): Three distinct return values: positive (remaining), -1 (no TTL), -2 (key missing).

   **PTTL** (`pub fn pttl(db: &mut Database, args: &[Frame]) -> Frame`):
   - Same as TTL but returns milliseconds: `remaining.as_millis() as i64`.

   **PERSIST** (`pub fn persist(db: &mut Database, args: &[Frame]) -> Frame`):
   - Arity: exactly 1.
   - If key exists and has TTL: remove expiry (set to None). Return `Frame::Integer(1)`.
   - If key exists but no TTL: return `Frame::Integer(0)`.
   - If key does not exist: return `Frame::Integer(0)`.

   **TYPE** (`pub fn type_cmd(db: &mut Database, args: &[Frame]) -> Frame`):
   - Arity: exactly 1.
   - If key exists: match on RedisValue variant. `RedisValue::String(_)` -> return `Frame::SimpleString(Bytes::from_static(b"string"))`.
   - If key does not exist: return `Frame::SimpleString(Bytes::from_static(b"none"))`.
   - Note: named `type_cmd` because `type` is a Rust keyword.

1. **src/command/key.rs** -- Add KEYS with a minimal Redis glob matcher and RENAME/RENAMENX:

   **Glob matcher** (private function `fn glob_match(pattern: &[u8], string: &[u8]) -> bool`):
   Implement a minimal Redis-compatible glob matcher supporting:
   - `*` -- matches any sequence of bytes (including empty)
   - `?` -- matches exactly one byte
   - `[abc]` -- matches any byte in the set
   - `[^abc]` or `[!abc]` -- matches any byte NOT in the set
   - `[a-z]` -- matches byte range (inclusive)
   - `\x` -- escapes special character (next byte is literal)

   Implementation approach: recursive or iterative matching. For each position in pattern:
   - `*`: try matching rest of pattern at every position in string (greedy or lazy). Iterative approach: advance string pointer, for each position try matching rest of pattern.
   - `?`: match one byte, advance both.
   - `[...]`: parse character class, match one byte against class, advance both.
   - `\`: skip backslash, match next pattern byte literally against string byte.
   - Other: literal byte match.

   This is ~50-80 lines. Do NOT use regex crate. Test with Redis documentation examples:
   - `h?llo` matches "hello", "hallo", "hxllo"
   - `h*llo` matches "hllo", "heeeello"
   - `h[ae]llo` matches "hello", "hallo" but not "hillo"
   - `h[^e]llo` matches "hallo", "hbllo" but not "hello"
   - `h[a-b]llo` matches "hallo", "hbllo" but not "hcllo"

   **KEYS** (`pub fn keys(db: &mut Database, args: &[Frame]) -> Frame`):
   - Arity: exactly 1 (pattern).
   - If pattern is `*`: return all keys.
   - Otherwise: iterate `db.keys()`, filter by `glob_match(pattern, key)`. But IMPORTANT: must perform lazy expiry check on each key before including it. This means we need a method that iterates keys while performing expiry checks. The simplest approach: collect all keys first, then for each key call `db.exists(key)` (which does lazy expiry), and if exists, include in results. Or: iterate db.keys() to get a Vec of all key Bytes, then for each key call db.get(key) to trigger lazy expiry, collect those that return Some.
   - Since db.keys() returns an iterator over &Bytes from the internal HashMap, and we need to call db.get() (which takes &mut self), we must collect keys first: `let all_keys: Vec<Bytes> = db.keys().cloned().collect();` then iterate and filter.
   - Return `Frame::Array(matching_keys.into_iter().map(|k| Frame::BulkString(k)).collect())`.

   **RENAME** (`pub fn rename(db: &mut Database, args: &[Frame]) -> Frame`):
   - Arity: exactly 2 (key, newkey).
   - If source key does not exist: return `ERR no such key`.
   - If source == destination: return OK immediately (Pitfall 5: no-op, don't delete).
   - Remove source entry. Set destination to that entry (overwriting if exists). Return OK.
   - IMPORTANT: preserve the entry's TTL by moving the entire Entry struct.

   **RENAMENX** (`pub fn renamenx(db: &mut Database, args: &[Frame]) -> Frame`):
   - Arity: exactly 2 (key, newkey).
   - If source does not exist: return `ERR no such key`.
   - If source == destination: return `Frame::Integer(0)` (same key, destination "exists").
   - If destination exists: return `Frame::Integer(0)`.
   - Remove source, set destination. Return `Frame::Integer(1)`.
