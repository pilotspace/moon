1. **src/server/listener.rs** -- Refactor for testability. Add `pub async fn run_with_shutdown(config: ServerConfig, shutdown: CancellationToken) -> anyhow::Result<()>` that takes the CancellationToken externally instead of creating one internally. The existing `pub async fn run(config)` wraps it by creating a token, spawning a ctrl_c signal handler that cancels it, then calling `run_with_shutdown`. This preserves the main.rs interface while enabling tests to control shutdown.

2. Create `tests/integration.rs` with integration tests that:

**Test helper: start_server()** -- A helper function that:
   - Uses `TcpListener::bind("127.0.0.1:0")` to get an OS-assigned port, extracts the port via `.local_addr()`, drops the listener.
   - Creates a `CancellationToken`.
   - Spawns the server in a background tokio task using `server::listener::run_with_shutdown(config, token.clone())`.
   - Waits a small delay (50-100ms) after spawn before returning.
   - Returns the port number and the CancellationToken for shutdown.

**Tests to implement (each starts its own server on a unique port):**

- **test_ping_pong** -- Connect redis client, send PING, verify PONG response.
  ```rust
  #[tokio::test]
  async fn test_ping_pong() {
      let (port, shutdown) = start_server().await;
      let client = redis::Client::open(format!("redis://127.0.0.1:{}/", port)).unwrap();
      let mut conn = client.get_multiplexed_tokio_connection().await.unwrap();
      let result: String = redis::cmd("PING").query_async(&mut conn).await.unwrap();
      assert_eq!(result, "PONG");
      shutdown.cancel();
  }
  ```

- **test_set_get_roundtrip** -- SET key value, GET key, verify value.

- **test_set_with_ex** -- SET key value EX 1, verify GET returns value, sleep 1.1s, verify GET returns nil.

- **test_set_nx_xx** -- SET key value NX when key exists returns nil. SET key value XX when key missing returns nil.

- **test_mget_mset** -- MSET k1 v1 k2 v2, MGET k1 k2 k3. Verify v1, v2, nil.

- **test_incr_decr** -- INCR counter (returns 1), INCR counter (returns 2), DECR counter (returns 1).

- **test_del_exists** -- SET key val, EXISTS key (returns 1), DEL key (returns 1), EXISTS key (returns 0).

- **test_expire_ttl** -- SET key val, EXPIRE key 100, TTL key returns positive, PERSIST key, TTL key returns -1.

- **test_keys_pattern** -- SET hello 1, SET hallo 2, SET hxllo 3, KEYS "h?llo" returns 3 matches, KEYS "h[ae]llo" returns 2.

- **test_concurrent_clients** -- Spawn 5 async tasks, each connects and does SET/GET on its own key. Verify no cross-contamination.

- **test_pipeline** -- Send SET and GET in a pipeline (redis crate supports this). Verify both responses correct and ordered.

- **test_rename** -- SET src val, RENAME src dst, GET dst returns val, GET src returns nil.

- **test_type_command** -- SET key val, TYPE key returns "string", TYPE missing returns "none".

- **test_select_database** -- SELECT 1, SET key val, SELECT 0, GET key returns nil, SELECT 1, GET key returns val.

Use the `redis` crate async API (`get_multiplexed_tokio_connection` or `get_tokio_connection`). For SELECT tests, use `get_tokio_connection` (non-multiplexed) since multiplexed connections may not support SELECT cleanly.
