However, ALL of these branches execute `break` or `continue` BEFORE reaching the `is_local` dispatch at line ~3603. This means Approach B (holding borrow across the full loop) is NOT viable, but holding it around just the local fast-path IS safe.

1. CALL refresh_now() ONCE BEFORE THE LOOP: Before the `for frame in frames` loop (after the `responses.clear(); remote_groups.clear();` block around line 3394), add:
   ```rust
   {
       let mut dbs = databases.borrow_mut();
       dbs[selected_db].refresh_now();
   }
   ```
   This single time refresh per batch is sufficient — sub-millisecond accuracy is not needed per-command. Remove the per-command `dbs[selected_db].refresh_now();` call from inside the loop (line ~3606).

2. RESTRUCTURE THE LOCAL FAST-PATH BLOCK (lines ~3603-3657): Currently there are TWO separate `databases.borrow_mut()` calls — one at line 3605 for dispatch, and one at line 3639 for wakeup. Merge them into a single borrow that covers the entire local block:

   ```rust
   if is_local {
       let mut dbs = databases.borrow_mut();
       dbs[selected_db].refresh_now();          // REMOVE (moved before loop)
       let result = dispatch(&mut dbs[selected_db], cmd, cmd_args, &mut selected_db, db_count);
       drop(dbs);                               // REMOVE this early drop

       let response = match result { ... };

       // AOF logging section (does not need dbs)
       if !matches!(response, Frame::Error(_)) { ... aof ... }

       // Wakeup section — currently re-borrows:
       if !matches!(response, Frame::Error(_)) {
           let needs_wake = ...;
           if needs_wake {
               if let Some(key) = ... {
                   let mut dbs = databases.borrow_mut();  // SECOND borrow — MERGE INTO OUTER
                   let mut reg = blocking_registry.borrow_mut();
                   ...wakeup calls...
               }
           }
       }
       responses.push(response);
   }
   ```

   With the merged borrow:
   ```rust
   if is_local {
       let mut dbs = databases.borrow_mut();
       // NO refresh_now here — already called once before the loop
       let result = dispatch(&mut dbs[selected_db], cmd, cmd_args, &mut selected_db, db_count);

       let response = match result {
           DispatchResult::Response(f) => f,
           DispatchResult::Quit(f) => {
               should_quit = true;
               f
           }
       };

       // AOF logging (does not need dbs, but holding borrow is fine — no await)
       if !matches!(response, Frame::Error(_)) {
           if let Some(ref tx) = aof_tx {
               if aof::is_write_command(cmd) {
                   let serialized = aof::serialize_command(&frame);
                   let _ = tx.try_send(AofMessage::Append(serialized));
               }
           }
       }

       // Wakeup — reuse the SAME `dbs` from outer scope (no re-borrow!)
       if !matches!(response, Frame::Error(_)) {
           let needs_wake = cmd.eq_ignore_ascii_case(b"LPUSH")
               || cmd.eq_ignore_ascii_case(b"RPUSH")
               || cmd.eq_ignore_ascii_case(b"LMOVE")
               || cmd.eq_ignore_ascii_case(b"ZADD");
           if needs_wake {
               if let Some(key) = cmd_args.first().and_then(|f| extract_bytes(f)) {
                   let mut reg = blocking_registry.borrow_mut();
                   if cmd.eq_ignore_ascii_case(b"LPUSH")
                       || cmd.eq_ignore_ascii_case(b"RPUSH")
                       || cmd.eq_ignore_ascii_case(b"LMOVE")
                   {
                       crate::blocking::wakeup::try_wake_list_waiter(
                           &mut reg, &mut dbs[selected_db], selected_db, &key,
                       );
                   } else {
                       crate::blocking::wakeup::try_wake_zset_waiter(
                           &mut reg, &mut dbs[selected_db], selected_db, &key,
                       );
                   }
               }
           }
       }

       drop(dbs);  // Single drop at end of local block
       responses.push(response);
   }
   ```

   - The borrow is held ONLY within the `if is_local { ... }` block, NOT across the entire for-frame loop

Add a frame count cap to the Monoio handler's parse loop, matching the Tokio handler's behavior.

1. Find the parse loop at approximately line 3375-3382:
   ```rust
   frames.clear();
   loop {
       match codec.decode_frame(&mut read_buf) {
           Ok(Some(frame)) => frames.push(frame),
           Ok(None) => break,
           Err(_) => return,
       }
   }
   ```

2. Add a batch limit check after pushing each frame:
   ```rust
   frames.clear();
   loop {
       match codec.decode_frame(&mut read_buf) {
           Ok(Some(frame)) => {
               frames.push(frame);
               if frames.len() >= 1024 {
                   break;
               }
           }
           Ok(None) => break,
           Err(_) => return,
       }
   }
   ```

This prevents unbounded frame accumulation which could cause latency spikes or OOM with aggressive pipelining. The Tokio handler already has this cap. Remaining unparsed data in read_buf will be processed in the next loop iteration.
