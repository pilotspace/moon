   - `#[derive(Debug, Clone, PartialEq)] pub enum EvictionPolicy { NoEviction, AllKeysLru, AllKeysLfu, AllKeysRandom, VolatileLru, VolatileLfu, VolatileRandom, VolatileTtl }`
   - `impl EvictionPolicy { pub fn from_str(s: &str) -> Self` -- match lowercase: "allkeys-lru" => AllKeysLru, etc., default NoEviction
   - `pub fn try_evict_if_needed(db: &mut Database, config: &RuntimeConfig) -> Result<(), Frame>` -- if config.maxmemory == 0 return Ok(()); while db.estimated_memory() > config.maxmemory { if policy == NoEviction return Err(OOM frame); if !evict_one(db, config) return Err(OOM frame); } Ok(())
   - OOM error frame: `Frame::Error(Bytes::from_static(b"OOM command not allowed when used memory > 'maxmemory'"))`
   - `fn evict_one(db: &mut Database, config: &RuntimeConfig) -> bool` -- parse policy, call appropriate evict function
   - `fn evict_one_lru(db: &mut Database, samples: usize, volatile_only: bool) -> bool` -- collect eligible keys (all or only those with expires_at.is_some()), sample `samples` random keys using `rand::seq::IndexedRandom::choose_multiple`, find one with oldest last_access, remove it. Apply lfu_decay before comparison for LFU variant.
   - `fn evict_one_lfu(db: &mut Database, samples: usize, lfu_decay_time: u64, volatile_only: bool) -> bool` -- sample keys, apply lfu_decay to get effective counter, evict lowest counter (break ties by oldest last_access)
   - `fn evict_one_random(db: &mut Database, volatile_only: bool) -> bool` -- pick one random key (filtered by volatile if needed), remove it
   - `fn evict_one_volatile_ttl(db: &mut Database, samples: usize) -> bool` -- sample keys with TTL, evict the one with soonest expires_at

4. **Wire eviction + CONFIG into connection handler** (src/server/connection.rs):
   - Add `runtime_config: Arc<RwLock<RuntimeConfig>>` parameter to `handle_connection`
   - Before the normal dispatch block, after the is_write check and AOF serialization, if `is_write` call `try_evict_if_needed` under the db lock using a read-lock on runtime_config. If it returns Err(oom_frame), send oom_frame and continue (skip dispatch).
   - Intercept "CONFIG" command in the connection handler (like AUTH/BGSAVE): extract subcommand (GET/SET). For GET: read-lock runtime_config and call config_get. For SET: write-lock runtime_config and call config_set. Send response, continue.
   - After successful dispatch of write commands, call `db.increment_version(key)` for the affected key. However, since dispatch consumes the frame and the key is inside the frame, extract the key BEFORE dispatch. For single-key write commands, extract args[1] as the key. Call `dbs[selected_db].increment_version(&key_bytes)` after dispatch if the response is not an error. This is a best-effort approach -- MULTI/EXEC (Plan 03) will handle version more precisely.
   - On every successful `get()` (read) access in dispatch handlers, the Database should touch_access. Since this requires modifying every read handler, instead add the touch to `Database::get()` and `Database::get_mut()` directly -- update last_access on every access in db.rs. Do this in Task 1's db.rs changes.

5. **Wire into listener** (src/server/listener.rs):
   - Create `let runtime_config = Arc::new(RwLock::new(config.to_runtime_config()));` after creating `Arc<ServerConfig>`
   - Pass `runtime_config.clone()` to each `handle_connection` call
   - Update `use` statements for `RwLock`

NOTE on version tracking: Rather than trying to extract the key from every write command before dispatch, a simpler approach is to increment version inside `Database::set()` and `Database::remove()` methods directly -- every mutation goes through these. In `set()`, set `entry.version = self.get_next_version(key)` where get_next_version checks if key exists and returns old_version+1 or 1. Actually even simpler: just track a per-database global version counter. But per-key version is needed for WATCH. The cleanest approach: in `Database::set()`, before inserting, check if old entry exists and copy its version+1 to new entry's version field. In `Database::remove()`, this is a delete so version changes are tracked by absence. WATCH will snapshot version at WATCH time and check at EXEC time -- if key was deleted, get_version returns 0 which differs from the snapshot, correctly aborting.
