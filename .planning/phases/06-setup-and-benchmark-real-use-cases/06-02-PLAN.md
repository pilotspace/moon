  <action>
Create `bench.sh` with `#!/usr/bin/env bash` and `set -euo pipefail`.

**Script structure:**

1. **Configuration variables at top:**
   - `PORT_RUST=6399` (our server port)
   - `PORT_REDIS=6400` (Redis baseline port)
   - `REQUESTS=100000`
   - `KEYSPACE=100000`
   - `TESTS="set,get,incr,lpush,rpush,lpop,rpop,sadd,hset,spop,zadd,zrangebyscore,mset"`
   - `CLIENTS=(1 10 50)`
   - `PIPELINES=(1 16 64)`
   - `DATASIZES=(3 256 1024 4096)`
   - `RESULTS_DIR` set to a temp directory created with `mktemp -d`

2. **Helper functions:**
   - `log()` -- prints timestamped messages to stderr
   - `check_command()` -- verifies redis-benchmark is installed, prints install instructions if not
   - `run_benchmark()` -- accepts port, clients, pipeline, datasize; runs `redis-benchmark -p $port -c $clients -P $pipeline -d $datasize -n $REQUESTS -r $KEYSPACE -t $TESTS --csv` and saves CSV output to `$RESULTS_DIR/result_${label}_c${clients}_p${pipeline}_d${datasize}.csv`. IMPORTANT: redis-benchmark `--csv` output includes columns for latency percentiles. The CSV header row contains: "test","rps","avg_latency_ms","min_latency_ms","p50_latency_ms","p95_latency_ms","p99_latency_ms","p99.9_latency_ms","max_latency_ms". These columns MUST be preserved in the CSV files for downstream parsing.
   - `measure_memory()` -- accepts PID; outputs RSS in KB using `ps -o rss= -p $PID`
   - `measure_cpu()` -- accepts PID; captures CPU utilization percentage using `ps -o %cpu= -p $PID`. Called before and after each benchmark run to get a CPU snapshot. Stores results to `$RESULTS_DIR/cpu_${label}.txt`.
   - `csv_to_markdown()` -- parses CSV files and produces markdown table rows. MUST extract and include latency percentile columns (p50, p95, p99, p99.9) from the CSV output. Output table columns: Command | Clients | Pipeline | Data Size | ops/sec | p50 (ms) | p99 (ms) | p99.9 (ms). For the comparison table variant, also include the baseline server columns.

3. **Main execution flow:**
   - Parse arguments: `--rust-only` (skip Redis baseline), `--redis-only` (skip rust-redis), `--output FILE` (default: BENCHMARK.md)
   - Check prerequisites (`redis-benchmark` must be installed)
   - **Phase A: Throughput and latency benchmarks**
     - For each server (rust-redis on PORT_RUST, optionally redis-server on PORT_REDIS):
       - Log which server is being benchmarked
       - Start the server in background, capture PID
       - For each combination of clients x pipelines x datasizes:
         - Capture CPU snapshot before: `measure_cpu $SERVER_PID`
         - Run `redis-benchmark` with `--csv` output
         - Capture CPU snapshot after: `measure_cpu $SERVER_PID`
         - Capture CSV to results dir
       - Kill server
   - **Phase B: Memory benchmarks**
     - Start rust-redis with `--port $PORT_RUST` in background, capture PID
     - Measure RSS at rest (after 2-second warmup)
     - Load 100K keys: `redis-benchmark -p $PORT_RUST -n 100000 -r 100000 -t set -q`
     - Sleep 1 second for RSS to stabilize
     - Measure RSS under load
     - Kill server
     - Repeat for redis-server if available
   - **Phase C: Persistence overhead benchmarks**
     - For each mode: no persistence, AOF everysec, AOF always:
       - Start rust-redis with appropriate flags (`--appendonly yes --appendfsync always` etc.) using a temp dir for persistence files
       - Run `redis-benchmark -p $PORT_RUST -c 50 -n 100000 -t set,get --csv`
       - Kill server, clean up temp persistence files
   - **Phase D: Generate BENCHMARK.md**
     - Write header with date, hardware info (`uname -a`, `sysctl -n hw.ncpu` on macOS or `nproc` on Linux, memory info)
     - Parse all CSV files from results dir, extracting ops/sec AND latency percentile columns (p50, p99, p99.9)
     - Generate markdown tables: one table per command, columns: Clients | Pipeline | Data Size | rust-redis ops/sec | p50 (ms) | p99 (ms) | p99.9 (ms) | Redis 7.x ops/sec | p50 | p99 | p99.9 | Throughput Ratio
     - CPU utilization section: table with Server | Benchmark Phase | Avg CPU % (parsed from cpu_*.txt files, computing average across snapshots)
     - Memory section: table with Server | RSS at rest | RSS with 100K keys
     - Persistence overhead section: table with Mode | SET ops/sec | GET ops/sec
     - Placeholder for "Top 3 Bottlenecks" analysis section (to be filled in Plan 03)

4. **Cleanup:** Remove temp results dir on EXIT trap.

The script manages server lifecycle for ALL benchmarks for full automation:
- Start rust-redis in background, wait for it to accept connections (`redis-cli -p $PORT ping` in a retry loop, max 5 seconds)
- Run benchmarks
- Kill rust-redis
- If not `--rust-only`: start redis-server in background, wait for connections, run benchmarks, kill redis-server

Make the script executable with `chmod +x bench.sh`.
