<objective>
Upgrade `handle_connection_sharded_monoio` from a simplified local-only dispatch to a full cross-shard-aware handler with AOF logging.

Currently the monoio handler calls `dispatch()` directly for every command, ignoring shard routing entirely. All parameters like `dispatch_tx`, `aof_tx`, `spsc_notifiers` are unused (underscore-prefixed). This means:
- MGET/MSET/DEL with keys on different shards return nulls/wrong counts
- DBSIZE/KEYS/SCAN only return local shard results
- Single-key SET on shard 0 for a key hashed to shard 2 stores it on shard 0 (wrong)
- No AOF or WAL logging happens

Purpose: Make `--shards 4` functionally correct under monoio for all data commands.
Output: Updated `handle_connection_sharded_monoio` with shard routing, cross-shard dispatch, and AOF logging.
</objective>

**What to port from the Tokio handler (handle_connection_sharded):**

1. **Primary key extraction and shard routing** (Tokio lines ~1700-1750):
   - For each parsed command, call `extract_primary_key(cmd, args)`
   - If key maps to a different shard via `key_to_shard(key, num_shards)`, dispatch via SPSC:
     ```rust
     let target = key_to_shard(key, num_shards);
     if target != shard_id {
         let (tx, rx) = channel::oneshot();
         let msg = ShardMessage::Execute {
             db_index: selected_db,
             command: std::sync::Arc::new(frame.clone()),
             reply_tx: tx,
         };
         // Push to SPSC with spin-retry
         let target_idx = ChannelMesh::target_index(shard_id, target);
         loop {
             let push_result = {
                 let mut producers = dispatch_tx.borrow_mut();
                 producers[target_idx].try_push(msg_pending)
             };
             match push_result {
                 Ok(()) => {
                     spsc_notifiers[target].notify_one();
                     break;
                 }
                 Err(val) => {
                     msg_pending = val;
                     monoio::time::sleep(std::time::Duration::from_micros(10)).await;
                 }
             }
         }
         let response = rx.await.unwrap_or(Frame::Error(Bytes::from_static(b"ERR cross-shard dispatch failed")));
         // encode response
         continue; // skip local dispatch
     }
     ```

2. **Multi-key command coordination** (Tokio lines ~2030-2080):
   - Before local dispatch, check for multi-key commands when num_shards > 1:
     ```rust
     if num_shards > 1 {
         let is_multi_key = cmd.eq_ignore_ascii_case(b"MGET")
             || cmd.eq_ignore_ascii_case(b"MSET")
             || (cmd.eq_ignore_ascii_case(b"DEL") && cmd_args.len() > 1)
             || (cmd.eq_ignore_ascii_case(b"UNLINK") && cmd_args.len() > 1)
             || (cmd.eq_ignore_ascii_case(b"EXISTS") && cmd_args.len() > 1);
         if is_multi_key {
             let response = coordinator::coordinate_multi_key(
                 cmd, cmd_args, shard_id, num_shards, selected_db,
                 &databases, &dispatch_tx,
             ).await;
             codec.encode_frame(&response, &mut write_buf);
             continue;
         }
     }
     ```

3. **Cross-shard aggregation commands** (Tokio lines ~2082-2130):
   - KEYS -> `coordinator::coordinate_keys(...).await`
   - SCAN -> `coordinator::coordinate_scan(...).await`
