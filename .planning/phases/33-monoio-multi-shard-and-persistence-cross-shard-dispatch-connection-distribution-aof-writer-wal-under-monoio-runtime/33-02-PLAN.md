Build the monoio release binary and run comprehensive cross-shard tests:

```bash
cargo build --release --no-default-features --features runtime-monoio

# Start 4-shard server with AOF enabled
mkdir -p /tmp/monoio-verify
./target/release/rust-redis --port 6479 --shards 4 --dir /tmp/monoio-verify --appendonly yes &
sleep 1

# 1. Basic SET/GET on various keys (will route to different shards)
redis-cli -p 6479 SET key1 val1
redis-cli -p 6479 SET key2 val2
redis-cli -p 6479 SET key3 val3
redis-cli -p 6479 SET key4 val4
redis-cli -p 6479 SET key5 val5
redis-cli -p 6479 SET key6 val6

# 2. Verify all GETs return correct values
redis-cli -p 6479 GET key1  # expect "val1"
redis-cli -p 6479 GET key3  # expect "val3"
redis-cli -p 6479 GET key6  # expect "val6"

# 3. Cross-shard MGET (keys distributed across shards by hash)
redis-cli -p 6479 MGET key1 key2 key3 key4 key5 key6
# expect: val1 val2 val3 val4 val5 val6

# 4. Cross-shard MSET
redis-cli -p 6479 MSET a 10 b 20 c 30 d 40
redis-cli -p 6479 MGET a b c d
# expect: 10 20 30 40

# 5. DBSIZE (aggregate across all shards)
redis-cli -p 6479 DBSIZE
# expect: (integer) 10

# 6. KEYS pattern (aggregate across all shards)
redis-cli -p 6479 KEYS '*'
# expect: 10 keys listed

# 7. Cross-shard DEL
redis-cli -p 6479 DEL key1 key2 key3
# expect: (integer) 3

# 8. INCR (should route to correct shard)
redis-cli -p 6479 INCR counter1
redis-cli -p 6479 INCR counter1
redis-cli -p 6479 GET counter1
# expect: "2"

# 9. AOF verification
ls -la /tmp/monoio-verify/

# 10. WAL verification (per-shard WAL files should have content)
wc -c /tmp/monoio-verify/shard-*.wal
```

If any test fails, diagnose the root cause and fix it. Common failure modes:
- MGET returns nulls: coordinator not being called, or dispatch_tx not wired
- DBSIZE unknown: handler not routing to coordinator
- Wrong shard storage: extract_primary_key not called before dispatch
- AOF empty: aof_tx not wired or is_write_command check missing

Fix any bugs found in `src/server/connection.rs` or `src/shard/coordinator.rs`.
