```rust
//! Per-shard Write-Ahead Log (WAL) writer.
//!
//! Each shard maintains its own WAL file containing RESP-encoded write commands.
//! Writes are buffered in memory and flushed/fsynced on a configurable interval
//! (default 1ms for batched fsync). On Linux, uses direct file I/O; on macOS,
//! uses std::fs as fallback.
//!
//! Replaces the global AOF writer from Phase 11 with per-shard locality.

use std::io::Write;
use std::path::{Path, PathBuf};
use std::time::Instant;

use bytes::Bytes;
use tracing::{error, info};
```

**WalWriter struct:**

```rust
/// Per-shard WAL writer with batched fsync.
///
/// Accumulates RESP-encoded command bytes between fsync intervals.
/// Flush is called explicitly by the shard event loop on its 1ms tick.
pub struct WalWriter {
    shard_id: usize,
    /// Buffered RESP bytes awaiting flush.
    buf: Vec<u8>,
    /// File handle for the WAL.
    file: Option<std::fs::File>,
    /// Path to the WAL file.
    file_path: PathBuf,
    /// Current write offset in the file.
    write_offset: u64,
    /// Timestamp of last fsync.
    last_fsync: Instant,
    /// Current snapshot epoch (for WAL truncation after snapshot).
    epoch: u64,
    /// Total bytes written since last truncation.
    bytes_written: u64,
}
```

**Key methods:**

1. `pub fn new(shard_id: usize, dir: &Path) -> std::io::Result<Self>`
   - WAL file path: `{dir}/shard-{shard_id}.wal`
   - Opens file in append+create mode
   - Initializes buf with 8KB capacity (pre-allocated)
   - Sets last_fsync to Instant::now(), epoch to 0

2. `pub fn append(&mut self, data: &[u8])`
   - Extends buf with data. No I/O here -- just buffering.
   - This is the hot path called on every write command.

3. `pub fn flush_if_needed(&mut self) -> std::io::Result<()>`
   - Called on the shard's 1ms tick (spsc_interval)
   - If buf is empty, return Ok(())
   - Write buf to file via file.write_all(&self.buf)
   - Call file.sync_data() for fsync
   - Update write_offset, bytes_written
   - Clear buf (but don't shrink allocation)
   - Update last_fsync

4. `pub fn flush_sync(&mut self) -> std::io::Result<()>`
   - Force flush regardless of timer. Used on shutdown and before snapshot.
   - Same as flush_if_needed but unconditional.

5. `pub fn truncate_after_snapshot(&mut self, new_epoch: u64) -> std::io::Result<()>`
   - Called after snapshot at new_epoch completes
   - Flush any pending data first
   - Close current file
   - Rename current WAL to .wal.old (backup)
   - Open a fresh WAL file at the same path
   - Reset write_offset, bytes_written, epoch to new_epoch
   - The .wal.old file can be cleaned up later or on next truncation

6. `pub fn shutdown(&mut self) -> std::io::Result<()>`
   - Flush any pending data, fsync, drop file handle

7. `pub fn path(&self) -> &Path` -- returns file_path reference

8. `pub fn bytes_written(&self) -> u64` -- for monitoring

**WAL replay function (standalone):**

```rust
/// Replay a per-shard WAL file into the given databases.
///
/// Uses the same RESP parsing and command dispatch as aof::replay_aof.
/// Returns the number of commands successfully replayed.
pub fn replay_wal(databases: &mut [Database], path: &Path) -> anyhow::Result<usize> {
    // Delegate to aof::replay_aof since the format is identical (RESP wire format)
    crate::persistence::aof::replay_aof(databases, path)
}
```

**WAL path helper:**

```rust
/// Construct the WAL file path for a given shard.
pub fn wal_path(dir: &Path, shard_id: usize) -> PathBuf {
    dir.join(format!("shard-{}.wal", shard_id))
}
```

**Important notes:**
- The WAL format is identical to AOF (RESP wire format). This is deliberate for compatibility.
- `is_write_command` and `serialize_command` from aof.rs are reused directly.
- `replay_wal` simply delegates to `replay_aof` since the format is the same.
- On Linux, a future optimization could use io_uring for file writes, but for now use std::fs::File which works on all platforms. The io_uring optimization is a refinement that can be added without API changes.
- Pre-allocate buf with 8KB capacity to avoid frequent reallocations during normal operation.
