/// Iterate over all occupied (key, value) pairs in this segment.
/// Returns references to initialized keys and values.
pub fn iter_occupied(&self) -> impl Iterator<Item = (&K, &V)> {
    (0..TOTAL_SLOTS).filter_map(move |slot| {
        if Self::is_full_ctrl(self.ctrl_byte(slot)) {
            // SAFETY: slot is FULL, so key and value are initialized.
            Some(unsafe { (self.keys[slot].assume_init_ref(), self.values[slot].assume_init_ref()) })
        } else {
            None
        }
    })
}
```

//! Forkless compartmentalized snapshot engine.
//!
//! Serializes DashTable segments one at a time as a cooperative task within the
//! shard event loop. Keys modified in not-yet-serialized segments have their old
//! values captured in a per-snapshot overflow buffer (segment-level COW).

use std::io::{Cursor, Read, Write};
use std::path::{Path, PathBuf};

use anyhow::{bail, Context};
use bytes::Bytes;
use crc32fast::Hasher;

use crate::storage::dashtable::DashTable;
use crate::storage::db::Database;
use crate::storage::entry::{current_secs, current_time_ms, Entry};
use crate::persistence::rdb;

// Per-shard snapshot format constants
const SHARD_RDB_MAGIC: &[u8] = b"RRDSHARD";
const SHARD_RDB_VERSION: u8 = 1;
const EOF_MARKER: u8 = 0xFF;
const SEGMENT_BLOCK_MARKER: u8 = 0xFD;
const DB_SELECTOR: u8 = 0xFE;
```

**SnapshotState struct:**

```rust
/// State machine for cooperative segment-by-segment snapshot.
///
/// Created when a snapshot epoch begins. Advanced one segment per tick.
/// Completed when all segments across all databases have been serialized.
pub struct SnapshotState {
    /// Snapshot epoch number.
    pub epoch: u64,
    /// Current database index being serialized (0..num_databases).
    current_db: usize,
    /// Current segment index within the current database.
    current_segment: usize,
    /// Total segment count for the current database (captured at epoch start).
    segments_in_current_db: usize,
    /// Total number of databases to snapshot.
    num_databases: usize,
    /// Segment counts per database, captured at epoch start.
    segment_counts: Vec<usize>,
    /// Base timestamps per database.
    base_timestamps: Vec<u32>,
    /// Output buffer accumulating serialized bytes.
    output_buf: Vec<u8>,
    /// COW overflow buffer: (segment_storage_idx, key, entry) for entries
    /// modified before their segment was serialized.
    overflow: Vec<(usize, Bytes, Entry)>,
    /// Set of segment storage indices that have already been serialized.
    /// Used to determine if a segment is still "pending" (not yet serialized).
    serialized_segments: Vec<bool>,
    /// Shard ID for the snapshot file header.
    shard_id: u16,
    /// Output file path.
    file_path: PathBuf,
    /// Whether the header has been written.
    header_written: bool,
}
```

**Key methods on SnapshotState:**

1. `pub fn new(shard_id: u16, epoch: u64, databases: &[Database], file_path: PathBuf) -> Self`
   - Captures segment_counts and base_timestamps from each database at epoch start
   - Initializes serialized_segments as vec![false; max_segment_count] where max is the largest segment_count across all DBs
   - Does NOT write header yet (deferred to first advance call)

2. `pub fn is_segment_pending(&self, db_index: usize, segment_storage_idx: usize) -> bool`
   - Returns true if this segment in this db has NOT been serialized yet AND the snapshot is active for this db
   - Simple check: db_index >= current_db OR (db_index == current_db AND segment_storage_idx >= current_segment... but actually we track by segment store idx in serialized_segments)
   - Better approach: `!self.serialized_segments[segment_storage_idx]` when db_index == current_db, or `true` when db_index > current_db

3. `pub fn capture_cow(&mut self, segment_storage_idx: usize, key: Bytes, old_entry: Entry)`
   - Pushes (segment_storage_idx, key, old_entry) onto overflow buffer

4. `pub fn advance_one_segment(&mut self, databases: &[Database]) -> bool`
   - If !header_written, write the header first:
     - SHARD_RDB_MAGIC (8 bytes) + version (1 byte) + shard_id (2 bytes LE) + epoch (8 bytes LE)
   - If current_db >= num_databases, return true (done)
   - Get current database, get segment at current_segment
   - Write DB_SELECTOR + db_index if this is the first segment of a new database
   - Write SEGMENT_BLOCK_MARKER + segment_idx (4 bytes LE) + entry_count (4 bytes LE)
   - Start a per-segment CRC32 hasher
   - First, drain overflow entries for this segment_storage_idx (they represent modified entries)
   - Track which keys were in overflow (to skip them from the live segment)
   - Then iterate segment's occupied slots via iter_occupied(), skip expired entries, skip keys already in overflow
   - Use rdb::write_entry for each entry (key, entry, base_ts)
   - Write per-segment CRC32 (4 bytes LE)
   - Mark segment as serialized: serialized_segments[current_segment] = true
   - Advance current_segment; if past segments_in_current_db, advance current_db, reset current_segment to 0, update segments_in_current_db
   - Return true if all databases done

5. `pub fn finalize(&mut self) -> anyhow::Result<()>`
   - Write EOF_MARKER
   - Compute global CRC32 of entire output_buf, append 4 bytes LE
   - Atomic write: write to .tmp, then rename to file_path

6. `pub fn is_complete(&self) -> bool`
   - current_db >= num_databases

**Standalone functions:**

`pub fn shard_snapshot_save(shard_id: u16, epoch: u64, databases: &[Database], path: &Path) -> anyhow::Result<()>`
  - Creates SnapshotState, runs advance_one_segment in a loop until complete, then finalizes
  - This is the synchronous all-at-once path for testing; the cooperative path uses advance_one_segment per tick

`pub fn shard_snapshot_load(databases: &mut [Database], path: &Path) -> anyhow::Result<usize>`
  - Reads SHARD_RDB_MAGIC header, verifies version
  - Reads shard_id, epoch (for logging/verification)
  - Loops over segment blocks: reads SEGMENT_BLOCK_MARKER, segment_idx, entry_count
  - For each entry: uses rdb::read_entry to deserialize
  - Verifies per-segment CRC32
  - Skips expired entries
  - Returns total keys loaded

**Important implementation details:**
- The overflow buffer is a simple `Vec<(usize, Bytes, Entry)>`. Since only one segment is pending serialization at a time, and segments have max 60 entries, overflow is bounded.
- For the COW intercept, the snapshot tracks which segment storage index maps to which segment. Since DashTable's directory can have multiple entries pointing to the same segment, use `segment_index_for_hash` (added in Task 1).
- The `serialized_segments` Vec is indexed by segment storage index (0..segment_count()), not by directory index.
- Make `is_segment_pending` and `capture_cow` cheap -- they run on every write during snapshot.
