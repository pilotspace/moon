Purpose: GET currently does: `as_bytes_owned()` (18ns Bytes alloc) -> `Frame::BulkString` (4ns) -> `serialize()` (20ns match+write) = 42ns. **Step 1: Add DirectGetResult to command/mod.rs**

The key challenge: GET needs to borrow from Database, but dispatch returns an owned value. Solution: split the GET fast path out of dispatch entirely. In connection.rs, detect GET before calling dispatch and handle it inline with direct database access.

Add to `src/command/mod.rs`:
```rust
/// Result of a direct GET operation — avoids Frame allocation.
pub enum DirectGetResult {
    /// Value bytes serialized directly (no Frame/Bytes intermediary)
    Serialized,
    /// Miss — key doesn't exist
    Miss,
    /// Wrong type — need to return WRONGTYPE error via Frame
    WrongType,
    /// Not a GET — fall through to normal dispatch
    NotGet,
}
```

**Step 2: Add direct_get_serialize to command/string.rs**

Create a function that takes Database + args + output BytesMut and serializes directly:

```rust
use bytes::BufMut;

/// Direct GET serialization: write RESP bulk string directly to output buffer.
/// Borrows value from Database (no Bytes allocation, no Frame construction).
/// Returns DirectGetResult indicating what happened.
pub fn direct_get_serialize(db: &mut Database, args: &[Frame], out: &mut BytesMut) -> super::DirectGetResult {
    if args.len() != 1 {
        return super::DirectGetResult::NotGet; // fall through to normal dispatch for error
    }
    let key = match extract_bytes(&args[0]) {
        Some(k) => k,
        None => return super::DirectGetResult::NotGet,
    };
    match db.get(key) {
        Some(entry) => match entry.value.as_bytes() {
            Some(data) => {
                // Serialize RESP bulk string directly: $<len>\r\n<data>\r\n
                out.put_u8(b'$');
                let mut itoa_buf = itoa::Buffer::new();
                out.put_slice(itoa_buf.format(data.len()).as_bytes());
                out.put_slice(b"\r\n");
                out.put_slice(data);
                out.put_slice(b"\r\n");
                super::DirectGetResult::Serialized
            }
            None => super::DirectGetResult::WrongType,
        },
        None => super::DirectGetResult::Miss,
    }
}
```

WAIT -- this approach still creates a Frame for the response. The real win requires bypassing the codec's Encoder entirely. But the Framed<TcpStream, RespCodec> abstraction forces all responses through `Encoder<Frame>`.

**Revised approach**: Instead of bypassing Frame entirely (which requires deep refactoring of the Framed abstraction), eliminate just the `as_bytes_owned()` allocation. The key insight: for the GET hot path, `as_bytes()` returns `&[u8]`, and we can create a `Bytes` from that without copying for heap strings:

Modify `string::get()` directly:
```rust
pub fn get(db: &mut Database, args: &[Frame]) -> Frame {
    if args.len() != 1 {
        return err_wrong_args("GET");
    }
    let key = match extract_bytes(&args[0]) {
        Some(k) => k,
        None => return err_wrong_args("GET"),
    };
    match db.get(key) {
        Some(entry) => match entry.value.as_bytes_owned() {
            Some(v) => Frame::BulkString(v),
            None => Frame::Error(Bytes::from_static(
                b"WRONGTYPE Operation against a key holding the wrong kind of value"
            )),
        },
        None => Frame::Null,
    }
}
```

Actually, `as_bytes_owned()` already does Arc bump (no memcpy) for heap strings and copy for inline strings (<=12B). The 18ns cost cited in OPTIMIZATION-ANALYSIS.md is for the 256B benchmark value which should be heap-allocated. Let me verify: 256B > 12B SSO threshold, so it's a heap Bytes, meaning `as_bytes_owned()` is just `self.heap_bytes.clone()` = Arc bump = ~5ns, not 18ns.

**Re-read the analysis**: The profiling says `as_bytes_owned` costs 54.5ns total. Looking at CompactValue, for heap strings `as_bytes_owned` does `.clone()` on Bytes which is an Arc bump. For 256B that should be fast. The 54ns might include the Entry lookup cost too.

Given this uncertainty, the most impactful approach is the **pre-serialized response** pattern. Instead of returning `Frame::BulkString(bytes)` which then gets serialized by the codec, pre-serialize the RESP directly:

Add a new Frame variant or use an existing mechanism. The codec already has `encode_frame` which calls `protocol::serialize()`. The simplest approach: add a `Frame::PreSerialized(Bytes)` variant that the codec writes directly without re-serialization.

```rust
// In src/protocol/frame.rs, add variant:
PreSerialized(Bytes),  // Already-serialized RESP data, write directly

// In protocol::serialize(), add branch:
Frame::PreSerialized(data) => dst.put_slice(data),
```

Then in `string::get()`:
```rust
pub fn get(db: &mut Database, args: &[Frame]) -> Frame {
    if args.len() != 1 { return err_wrong_args("GET"); }
    let key = match extract_bytes(&args[0]) {
        Some(k) => k,
        None => return err_wrong_args("GET"),
    };
    match db.get(key) {
        Some(entry) => match entry.value.as_bytes() {
            Some(data) => {
                // Pre-serialize RESP bulk string: skip Bytes alloc + Frame::serialize overhead
                let mut buf = BytesMut::with_capacity(data.len() + 16);
                buf.put_u8(b'$');
                let mut itoa_buf = itoa::Buffer::new();
                buf.put_slice(itoa_buf.format(data.len()).as_bytes());
                buf.put_slice(b"\r\n");
                buf.put_slice(data);  // single memcpy from CompactValue
                buf.put_slice(b"\r\n");
                Frame::PreSerialized(buf.freeze())
            }
            None => Frame::Error(Bytes::from_static(
                b"WRONGTYPE Operation against a key holding the wrong kind of value"
            )),
        },
        None => Frame::Null,
    }
}
```

This eliminates: (1) `as_bytes_owned()` Bytes alloc, (2) Frame::BulkString match in serialize(), (3) itoa + length prefix re-computation in serialize(). Single memcpy directly from CompactValue's backing store.
