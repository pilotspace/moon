<objective>
OPT-6: Zero-copy argument slicing in RESP parser.

Purpose: The current `parse_single_frame()` uses `Bytes::copy_from_slice()` for every bulk string (~18ns each). For arguments >= 10 bytes (keys, values), `Bytes::slice()` on a frozen buffer costs only ~5ns (Arc refcount bump). This saves ~13ns per key/value argument.

Output: Modified `parse()` function using two-pass approach: validate with `parse_single_frame` (no allocs), then `buf.split_to(pos).freeze()` + `parse_frame_zerocopy()` for zero-copy frame extraction.
</objective>

Concrete change to `parse()`:
```rust
pub fn parse(buf: &mut BytesMut, config: &ParseConfig) -> Result<Option<Frame>, ParseError> {
    if buf.is_empty() {
        return Ok(None);
    }

    match buf[0] {
        b'+' | b'-' | b':' | b'$' | b'*'
        | b'%' | b'~' | b',' | b'#' | b'_' | b'=' | b'(' | b'>'
        => { /* fall through to RESP parsing below */ }
        _ => return inline::parse_inline(buf),
    }

    // Pass 1: Validate structure and compute byte length (no allocations)
    let mut pos = 0;
    match parse_single_frame(&buf[..], &mut pos, config, 0) {
        Ok(_discard) => {
            // Pass 2: Zero-copy extraction from frozen Bytes
            let frozen = buf.split_to(pos).freeze();
            let mut zc_pos = 0;
            let frame = parse_frame_zerocopy(&frozen, &mut zc_pos, config, 0);
            Ok(Some(frame))
        }
        Err(ParseError::Incomplete) => Ok(None),
        Err(e) => Err(e),
    }
}
```

This leverages the existing `parse_frame_zerocopy` function (line 228) which already does `buf.slice()` for all bulk strings — zero-copy via Arc refcount bump.

The validation pass (`parse_single_frame`) still creates Frame objects but they are immediately discarded. The cost of the validation pass is just the integer parsing + bounds checking (~15ns), which is paid back by eliminating N x copy_from_slice calls (~18ns each for keys/values).

**Do NOT change** `parse_single_frame` or `parse_frame_zerocopy` — they already work correctly for their respective roles.
