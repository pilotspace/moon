<objective>
OPT-7: Slab segment allocator for DashTable.

Purpose: Each DashTable segment is individually heap-allocated via `Box::new(Segment::new(...))`, paying jemalloc per-allocation metadata (~16-32B per segment). At scale (1M keys, ~17K segments), this adds ~25B/key in RSS overhead. Pre-allocating segments in contiguous Vec slabs eliminates per-segment allocator metadata, improves cache locality during splits, and reduces memory fragmentation.

Output: `SegmentSlab` struct managing contiguous slabs of segments, replacing `Vec<Box<Segment>>` in DashTable.
</objective>

**Step 1: Create SegmentSlab struct** (add above DashTable definition)

```rust
/// Slab allocator for DashTable segments.
///
/// Pre-allocates segments in contiguous Vec "slabs" instead of individual Box allocations.
/// Benefits: eliminates per-segment allocator metadata (~16-32B/segment), improves cache
/// locality, reduces fragmentation.
///
/// Segments are addressed by a flat index. The slab grows by allocating new contiguous
/// blocks (doubling strategy) when the current slab is full.
struct SegmentSlab<K, V> {
    /// Contiguous blocks of segments. Each slab is a Vec of initialized segments.
    slabs: Vec<Vec<Segment<K, V>>>,
    /// Free list of (slab_idx, slot_idx) pairs for reuse after hypothetical segment merging.
    /// Currently unused since DashTable only grows, but included for future merge support.
    free_list: Vec<(usize, usize)>,
    /// Number of segments per new slab allocation.
    slab_capacity: usize,
    /// Total number of allocated segments.
    total: usize,
    /// Mapping from flat index to (slab_idx, slot_idx).
    /// This avoids division/modulo on every access since slabs may have different sizes.
    index_map: Vec<(usize, usize)>,
}

impl<K, V> SegmentSlab<K, V> {
    fn new() -> Self {
        SegmentSlab {
            slabs: Vec::new(),
            free_list: Vec::new(),
            slab_capacity: 16, // Initial slab size, doubles on growth
            total: 0,
            index_map: Vec::new(),
        }
    }

    fn with_capacity(cap: usize) -> Self {
        let mut slab = Self::new();
        if cap > 0 {
            slab.slab_capacity = cap;
        }
        slab
    }

    /// Add a segment, returning its flat index.
    fn push(&mut self, segment: Segment<K, V>) -> usize {
        // Check if we can reuse a free slot
        if let Some((slab_idx, slot_idx)) = self.free_list.pop() {
            self.slabs[slab_idx][slot_idx] = segment;
            let flat_idx = self.index_map.iter().position(|&(s, sl)| s == slab_idx && sl == slot_idx)
                .expect("free_list entry must have index_map entry");
            return flat_idx;
        }

        // Check if current last slab has room
        let needs_new_slab = self.slabs.is_empty()
            || self.slabs.last().unwrap().len() >= self.current_slab_capacity();

        if needs_new_slab {
            let new_cap = if self.slabs.is_empty() {
                self.slab_capacity
            } else {
                // Double the slab size, cap at 1024
                (self.slabs.last().unwrap().capacity() * 2).min(1024)
            };
            self.slabs.push(Vec::with_capacity(new_cap));
        }

        let slab_idx = self.slabs.len() - 1;
        let slot_idx = self.slabs[slab_idx].len();
        self.slabs[slab_idx].push(segment);

        let flat_idx = self.total;
        self.index_map.push((slab_idx, slot_idx));
        self.total += 1;
        flat_idx
    }

    fn current_slab_capacity(&self) -> usize {
        self.slabs.last().map(|s| s.capacity()).unwrap_or(self.slab_capacity)
    }

    #[inline]
    fn len(&self) -> usize {
        self.total
    }

    #[inline]
    fn get(&self, idx: usize) -> &Segment<K, V> {
        let (slab_idx, slot_idx) = self.index_map[idx];
        &self.slabs[slab_idx][slot_idx]
    }

    #[inline]
    fn get_mut(&mut self, idx: usize) -> &mut Segment<K, V> {
        let (slab_idx, slot_idx) = self.index_map[idx];
        &mut self.slabs[slab_idx][slot_idx]
    }

    fn iter(&self) -> impl Iterator<Item = &Segment<K, V>> {
        self.index_map.iter().map(move |&(si, sli)| &self.slabs[si][sli])
    }

    fn iter_mut(&mut self) -> impl Iterator<Item = &mut Segment<K, V>> {
        // For iter_mut, we need to collect slab references to avoid borrow issues.
        // Since segments don't move, we can use raw pointer tricks safely.
        // But simpler: use index-based iteration.
        let index_map = self.index_map.clone();
        let slabs_ptr = &mut self.slabs as *mut Vec<Vec<Segment<K, V>>>;
        index_map.into_iter().map(move |(si, sli)| {
            unsafe { &mut (*slabs_ptr)[si][sli] }
        })
    }
}
```

- `iter_mut()`: collect from `self.segments.iter_mut()`

**Key concern**: The `split_segment` method calls `self.segments[seg_store_idx].split()` which needs mutable access to one segment while also needing to push a new segment. With `Vec<Box<>>` this works because push doesn't invalidate existing elements. With SegmentSlab this also works because `push` appends to a different slab or at the end of the current one — existing segment references remain valid. BUT: we must not hold a mutable reference to the segment while calling push. Extract the split result first:

```rust
fn split_segment(&mut self, dir_idx: usize) {
    let seg_store_idx = self.directory[dir_idx];
    let hasher = |k: &CompactKey| hash_key(k.as_ref());
    let new_seg = self.segments.get_mut(seg_store_idx).split(&hasher);
    let new_depth = new_seg.depth();
    let new_store_idx = self.segments.push(new_seg);
    // ... rest unchanged
}
```

This is safe because `split()` returns an owned `Segment` and the mutable borrow is released before `push()`.
