<objective>
Implement listpack (compact byte-array) and intset (sorted integer array) data structures for small collection encoding.

Purpose: Small collections (< 128 elements) waste significant memory in HashMap/HashSet/VecDeque due to per-node allocation overhead. Listpack stores entries in a contiguous byte array with ~2 bytes overhead per entry. Intset stores sorted integers at the minimum required width (2/4/8 bytes per integer).

Output: `src/storage/listpack.rs` and `src/storage/intset.rs` -- fully tested standalone modules.
</objective>

Create `src/storage/listpack.rs` following the Redis 7+ listpack format:

**Format:**
```
[total_bytes: u32 LE][num_elements: u16 LE][entries...][0xFF terminator]
```

**Entry format:**
```
[encoding_byte(s)][data][backlen]
```

**Encoding types (from Redis listpack spec):**
```rust
// 0xxxxxxx          -> 7-bit unsigned int (0-127), 1 byte encoding
// 10xxxxxx          -> 6-bit string length (0-63), followed by string bytes
// 110xxxxx + 1 byte -> 13-bit signed int
// 1110xxxx + 2 bytes -> string with 12-bit length (0-4095)
// 11110001 + 2 bytes -> 16-bit signed int
// 11110010 + 3 bytes -> 24-bit signed int
// 11110011 + 4 bytes -> 32-bit signed int
// 11110100 + 8 bytes -> 64-bit signed int
// 11110000 + 4 bytes -> string with 32-bit length
// 11111111          -> terminator (EOF)
```

**Backlen:** Variable-length encoding of the total entry size (encoding + data). Each backlen byte uses 7 data bits + 1 high bit (set = more bytes follow). Read backward from the byte before the next entry's encoding byte.

**Public API:**
```rust
pub struct Listpack {
    data: Vec<u8>,
}

impl Listpack {
    pub fn new() -> Self;
    pub fn len(&self) -> usize;
    pub fn is_empty(&self) -> bool;
    pub fn total_bytes(&self) -> usize;

    // Append/prepend
    pub fn push_back(&mut self, value: &[u8]);
    pub fn push_front(&mut self, value: &[u8]);

    // Positional access (O(N) scan)
    pub fn get_at(&self, index: usize) -> Option<ListpackEntry>;
    pub fn remove_at(&mut self, index: usize) -> bool;
    pub fn replace_at(&mut self, index: usize, value: &[u8]);

    // Search (O(N) scan)
    pub fn find(&self, value: &[u8]) -> Option<usize>;

    // Iteration
    pub fn iter(&self) -> ListpackIter<'_>;
    pub fn iter_rev(&self) -> ListpackRevIter<'_>;

    // Hash-specific: iterate as (field, value) pairs
    pub fn iter_pairs(&self) -> ListpackPairIter<'_>;

    // Conversion helpers
    pub fn to_vec(&self) -> Vec<Bytes>;
    pub fn to_hash_map(&self) -> HashMap<Bytes, Bytes>;
    pub fn to_hash_set(&self) -> HashSet<Bytes>;
    pub fn to_vec_deque(&self) -> VecDeque<Bytes>;

    // Sorted set pair iteration: (score_bytes, member_bytes) alternating
    pub fn iter_score_member_pairs(&self) -> ListpackPairIter<'_>;

    pub fn estimate_memory(&self) -> usize;
}

#[derive(Debug, Clone)]
pub enum ListpackEntry {
    Integer(i64),
    String(Vec<u8>),
}

impl ListpackEntry {
    pub fn as_bytes(&self) -> Vec<u8>;  // serialize to Redis-format bytes
}
```

**Internal functions:**
- `encode_entry(value: &[u8]) -> Vec<u8>` -- encode value + compute backlen
- `decode_entry_at(data: &[u8], pos: usize) -> (ListpackEntry, usize)` -- returns (entry, next_pos)
- `decode_entry_backward(data: &[u8], pos: usize) -> (ListpackEntry, usize)` -- returns (entry, prev_entry_start)
- `encode_backlen(entry_len: usize) -> Vec<u8>` -- variable-length backward pointer
- `decode_backlen(data: &[u8], pos: usize) -> (usize, usize)` -- returns (entry_len, backlen_size)
- `try_encode_as_integer(value: &[u8]) -> Option<i64>` -- attempt to parse as integer for compact encoding

**Auto-detection:** When `push_back` receives bytes that represent a valid integer string (e.g., b"42"), encode it as an integer entry for space savings.

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum IntsetEncoding {
    Int16 = 2,
    Int32 = 4,
    Int64 = 8,
}

#[derive(Debug, Clone)]
pub struct Intset {
    encoding: IntsetEncoding,
    data: Vec<u8>,  // packed sorted integers at encoding width
}

impl Intset {
    pub fn new() -> Self;
    pub fn len(&self) -> usize;
    pub fn is_empty(&self) -> bool;
    pub fn encoding(&self) -> IntsetEncoding;

    pub fn insert(&mut self, value: i64) -> bool;       // true if new
    pub fn remove(&mut self, value: i64) -> bool;        // true if existed
    pub fn contains(&self, value: i64) -> bool;
    pub fn get(&self, index: usize) -> Option<i64>;     // by sorted position

    pub fn iter(&self) -> IntsetIter<'_>;
    pub fn random(&self) -> Option<i64>;                 // random element

    pub fn to_hash_set(&self) -> HashSet<Bytes>;        // convert to string-encoded HashSet

    pub fn estimate_memory(&self) -> usize;
}
```

**Internal mechanics:**
- `encoding_for_value(value: i64) -> IntsetEncoding` -- minimum encoding width for a value
- `read_at(data: &[u8], encoding: IntsetEncoding, index: usize) -> i64` -- read integer at position
- `write_at(data: &mut [u8], encoding: IntsetEncoding, index: usize, value: i64)` -- write integer at position
- `binary_search(&self, value: i64) -> Result<usize, usize>` -- find position, Err(insert_pos) if not found
- `upgrade_encoding(&mut self, new_encoding: IntsetEncoding)` -- widen all values in-place

**Encoding upgrade:** When inserting a value that requires a wider encoding:
1. Allocate new Vec with wider width * existing count + 1
2. Copy all existing values from old encoding to new (iterate backward to avoid overwrite issues)
3. Insert the new value at the correct sorted position

**Binary search:** Read values at encoding width, compare as i64.
