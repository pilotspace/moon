<objective>
Implement a custom B+ tree data structure for sorted set score-ordered storage, replacing std BTreeMap.

Purpose: BTreeMap uses ~37 bytes overhead per entry (2 pointers + node metadata). A custom B+ tree with arena-allocated nodes and configurable fanout achieves ~3 bytes amortized overhead per entry -- a 12x memory reduction for sorted sets.

Output: `src/storage/bptree.rs` -- a fully tested, standalone B+ tree module with insert, remove, lookup, range iteration, reverse iteration, rank queries, and length tracking.
</objective>

Create `src/storage/bptree.rs` with the following design:

**Constants:**
```rust
const INTERNAL_FANOUT: usize = 16;  // 16 separator keys, 17 children -- fits ~256 bytes
const LEAF_CAPACITY: usize = 14;    // 14 (f64, Bytes) entries per leaf
```

**Types:**
```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub struct NodeId(u32);

#[derive(Debug, Clone)]
pub struct BPTree {
    root: NodeId,
    nodes: Vec<Node>,
    free_list: Vec<NodeId>,
    len: usize,
    height: usize,
    leaf_head: NodeId,  // first leaf for full iteration
}

#[derive(Debug, Clone)]
enum Node {
    Internal(InternalNode),
    Leaf(LeafNode),
}

#[derive(Debug, Clone)]
struct InternalNode {
    len: u16,
    keys: [(OrderedFloat<f64>, Bytes); INTERNAL_FANOUT],  // separator (score, member) pairs
    children: [NodeId; INTERNAL_FANOUT + 1],
}

#[derive(Debug, Clone)]
struct LeafNode {
    len: u16,
    entries: [(OrderedFloat<f64>, Bytes); LEAF_CAPACITY],
    next: Option<NodeId>,
    prev: Option<NodeId>,
}
```

Use `MaybeUninit` arrays or initialize with defaults for the fixed-size arrays. Entries within leaves and internal nodes are sorted by (score, member) using OrderedFloat for total ordering.

**Public API -- must match what sorted_set.rs needs:**
```rust
impl BPTree {
    pub fn new() -> Self;
    pub fn len(&self) -> usize;
    pub fn is_empty(&self) -> bool;
    pub fn clear(&mut self);

    // Core mutations
    pub fn insert(&mut self, score: OrderedFloat<f64>, member: Bytes) -> bool;  // true if new
    pub fn remove(&mut self, score: OrderedFloat<f64>, member: &[u8]) -> bool;  // true if existed

    // Lookups
    pub fn contains(&self, score: OrderedFloat<f64>, member: &[u8]) -> bool;

    // Range queries (inclusive bounds) -- returns iterator over (score, &member)
    pub fn range<'a>(&'a self, min: OrderedFloat<f64>, max: OrderedFloat<f64>) -> BPTreeIter<'a>;
    pub fn range_rev<'a>(&'a self, min: OrderedFloat<f64>, max: OrderedFloat<f64>) -> BPTreeRevIter<'a>;

    // Rank queries (0-based)
    pub fn rank(&self, score: OrderedFloat<f64>, member: &[u8]) -> Option<usize>;
    pub fn rev_rank(&self, score: OrderedFloat<f64>, member: &[u8]) -> Option<usize>;

    // Full iteration
    pub fn iter(&self) -> BPTreeIter<'_>;
    pub fn iter_rev(&self) -> BPTreeRevIter<'_>;

    // Index access (for ZRANGE by rank)
    pub fn get_by_rank(&self, rank: usize) -> Option<(OrderedFloat<f64>, &Bytes)>;
    pub fn range_by_rank(&self, start: usize, end: usize) -> Vec<(OrderedFloat<f64>, &Bytes)>;
}
```

**Internal mechanics:**
- `find_leaf(score, member)` -- descend from root using binary search within internal nodes
- `split_leaf(leaf_id)` -- split full leaf into two, promote median to parent
- `split_internal(node_id)` -- split full internal node, promote median to parent
- Node allocation via `alloc_leaf()`/`alloc_internal()` using free_list or Vec::push
- `merge_or_redistribute()` on remove when node drops below minimum occupancy (LEAF_CAPACITY/2)

**Rank computation:** Walk the tree counting entries in skipped subtrees. For `rank()`, traverse from root, at each internal node sum the counts of children to the left of the target path. For leaves, count entries before the target. This requires storing subtree counts in internal nodes -- add a `counts: [u32; INTERNAL_FANOUT + 1]` array to InternalNode.

**Iterator structs:**
```rust
pub struct BPTreeIter<'a> {
    tree: &'a BPTree,
    leaf: Option<NodeId>,
    index: usize,
    max: OrderedFloat<f64>,
}

pub struct BPTreeRevIter<'a> {
    tree: &'a BPTree,
    leaf: Option<NodeId>,
    index: usize,  // counts down from len-1
    min: OrderedFloat<f64>,
}
```

Both implement `Iterator<Item = (OrderedFloat<f64>, &'a Bytes)>`.

Add `pub mod bptree;` to `src/storage/mod.rs`.

IMPORTANT: The BPTree comparison key is `(OrderedFloat<f64>, Bytes)` -- score first, then member for tie-breaking. This matches Redis semantics where equal scores are ordered lexicographically by member.

For the fixed-size arrays, use a helper:

```rust
fn default_entries() -> [(OrderedFloat<f64>, Bytes); LEAF_CAPACITY] {
    std::array::from_fn(|_| (OrderedFloat(0.0), Bytes::new()))
}
```
