**Step 2: Update `zadd_member` helper.** Change the `scores` parameter from `&mut BTreeMap<(OrderedFloat<f64>, Bytes), ()>` to `&mut BPTree`:
```rust
fn zadd_member(
    members: &mut HashMap<Bytes, f64>,
    scores: &mut BPTree,
    member: Bytes,
    score: f64,
) -> bool {
    let is_new = if let Some(old_score) = members.remove(&member) {
        scores.remove(OrderedFloat(old_score), &member);
        false
    } else {
        true
    };
    members.insert(member.clone(), score);
    scores.insert(OrderedFloat(score), member);
    is_new
}
```

**Step 3: Update `zrem_member` helper.** Same type change:
```rust
fn zrem_member(
    members: &mut HashMap<Bytes, f64>,
    scores: &mut BPTree,
    member: &[u8],
) -> bool {
    if let Some(score) = members.remove(member) {
        scores.remove(OrderedFloat(score), member);
        true
    } else {
        false
    }
}
```

**Step 4: Update ALL command functions** that call `db.get_or_create_sorted_set()` or `db.get_sorted_set()`. The returned `scores` is now `&mut BPTree` / `&BPTree` instead of `&mut BTreeMap` / `&BTreeMap`.

Key API mapping (BTreeMap -> BPTree):

| BTreeMap operation | BPTree equivalent |
|---|---|
| `scores.range((min_key)..(max_key))` | `scores.range(min_score, max_score)` then filter by member bounds if needed |
| `scores.iter()` | `scores.iter()` -- yields `(OrderedFloat<f64>, &Bytes)` |
| `scores.len()` | `scores.len()` |
| `scores.iter().rev()` | `scores.iter_rev()` |
| `scores.range(..).count()` | iterate and count, or use rank difference |
| `scores.iter().nth(n)` | `scores.get_by_rank(n)` |
| `scores.insert((OrderedFloat(score), member), ())` | `scores.insert(OrderedFloat(score), member)` |
| `scores.remove(&(OrderedFloat(score), member))` | `scores.remove(OrderedFloat(score), &member)` |

**Commands that need BTreeMap->BPTree migration (all functions in sorted_set.rs):**

For each command, the pattern is:
1. Change destructuring from `(members, scores)` where scores was BTreeMap to where scores is BPTree
2. Replace BTreeMap range queries with BPTree range/range_rev calls
3. Replace BTreeMap rank computation (counting iter elements) with BPTree rank() calls
4. Replace BTreeMap insertion/removal with BPTree insert/remove

Specific commands and their key changes:

- **ZADD**: calls zadd_member (already updated)
- **ZREM**: calls zrem_member (already updated)
- **ZSCORE**: uses `members.get()` only -- no BTreeMap, no change needed
- **ZCARD**: uses `members.len()` -- no change needed
- **ZRANK**: currently counts elements in BTreeMap iteration. Replace with `scores.rank(OrderedFloat(score), member)`.
- **ZREVRANK**: replace with `scores.rev_rank(OrderedFloat(score), member)`
- **ZRANGE**: currently collects from BTreeMap iter with skip/take. Replace with `scores.range_by_rank(start, stop)` or iterate `scores.iter()` and skip/take. For ZRANGE BYSCORE, use `scores.range(min, max)`. For ZRANGE BYLEX, use `scores.range()` then filter by lex bounds.
- **ZRANGEBYSCORE**: use `scores.range(min_score, max_score)` then apply exclusive bounds and LIMIT
- **ZREVRANGEBYSCORE**: use `scores.range_rev(min_score, max_score)` then apply exclusive bounds and LIMIT
- **ZRANGEBYLEX**: all members have the same score; use `scores.range(OrderedFloat(f64::NEG_INFINITY), OrderedFloat(f64::INFINITY))` and filter by lex bounds
- **ZCOUNT**: `scores.range(min, max)` then count with bound filtering
- **ZLEXCOUNT**: similar to ZRANGEBYLEX but count only
- **ZRANGESTORE**: combines ZRANGE logic with store
- **ZINCRBY**: uses zadd_member (already updated)
- **ZPOPMIN/ZPOPMAX**: get first/last from BPTree via `scores.iter().next()` / `scores.iter_rev().next()`, then remove
- **ZRANDMEMBER**: uses members HashMap only -- no BTreeMap, no change
- **ZUNIONSTORE/ZINTERSTORE**: these clone source data to temporary HashMaps. The scores BTreeMap was used for iteration -- replace with BPTree iter(). The temporary result is built via zadd_member which is already updated.
- **ZSCAN**: iterates members HashMap -- no BTreeMap, no change
- **ZMSCORE**: uses members.get() -- no change
- **ZDIFF/ZDIFFSTORE**: iterates scores for ordering, replace with BPTree iter()
- **ZMPOP**: similar to ZPOPMIN/ZPOPMAX
