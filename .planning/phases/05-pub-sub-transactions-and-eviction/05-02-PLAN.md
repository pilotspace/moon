2. **Add subscriber mode to connection handler** (src/server/connection.rs):
   - Add parameter: `pubsub_registry: Arc<Mutex<PubSubRegistry>>`
   - Add connection-local state variables after `authenticated`:
     ```rust
     let mut subscription_count: usize = 0;
     let mut pubsub_rx: Option<mpsc::Receiver<Frame>> = None;
     let mut subscriber_id: u64 = 0;
     ```
   - The connection handler needs TWO modes. When `subscription_count > 0`, the client is in subscriber mode and uses a different tokio::select! branch.

   **Subscriber mode** (when subscription_count > 0):
   - Replace the main loop body with a check: `if subscription_count > 0 { /* subscriber loop */ } else { /* normal loop */ }`
   - In subscriber mode, use `tokio::select!` with THREE branches:
     - `result = framed.next()` -- read commands from client. Only allow SUBSCRIBE, UNSUBSCRIBE, PSUBSCRIBE, PUNSUBSCRIBE, PING, QUIT. Any other command returns `Frame::Error("ERR Can't execute '...' command while in subscriber mode")`.
     - `Some(msg) = async { pubsub_rx.as_mut().unwrap().recv().await }` -- forward published message to client via framed.send(msg). (Use a helper: `let rx = pubsub_rx.as_mut().unwrap(); rx.recv()`)
     - `_ = shutdown.cancelled()` -- break
   - For SUBSCRIBE in subscriber mode: for each channel arg, lock registry, call subscribe(channel, subscriber), increment subscription_count. Send subscribe_response(channel, subscription_count).
   - For UNSUBSCRIBE: if no args, unsubscribe_all. For each channel, lock registry, call unsubscribe(channel, sub_id), decrement subscription_count. Send unsubscribe_response. If subscription_count reaches 0, exit subscriber mode (pubsub_rx stays but subscriber mode loop exits).
   - PSUBSCRIBE/PUNSUBSCRIBE: same pattern with pattern variants.
   - PING in subscriber mode: respond with `Frame::Array(vec![BulkString("pong"), BulkString("")])` per Redis spec (not SimpleString).

   **Normal mode -- handle SUBSCRIBE/PSUBSCRIBE entering subscriber mode:**
   - In the authenticated command handling section (after AUTH/BGSAVE/BGREWRITEAOF intercepts), add intercepts for SUBSCRIBE, PSUBSCRIBE:
     - If "SUBSCRIBE" or "PSUBSCRIBE": allocate subscriber if not yet done: `let (tx, rx) = mpsc::channel::<Frame>(256); subscriber_id = next_subscriber_id(); pubsub_rx = Some(rx);`
     - Create `Subscriber::new(tx.clone(), subscriber_id)` (keep tx around for further subscribes)
     - For each channel/pattern arg: lock registry, subscribe/psubscribe, increment subscription_count, send response frame
     - Continue (the next loop iteration will enter subscriber mode)

   **Normal mode -- handle PUBLISH:**
   - Intercept "PUBLISH" in the authenticated section: extract channel (args[0]) and message (args[1]). Lock registry, call publish(&channel, &message). Send Frame::Integer(count). Continue.
   - PUBLISH is NOT a write command for AOF purposes (Pub/Sub messages not persisted per user decision).

   **Normal mode -- handle UNSUBSCRIBE/PUNSUBSCRIBE when not subscribed:**
   - If subscription_count == 0 and command is UNSUBSCRIBE/PUNSUBSCRIBE, send unsubscribe_response/punsubscribe_response with count 0. Continue.

   **Cleanup on disconnect:**
   - Before the connection handler returns (after the loop), if subscriber_id != 0, lock registry and call unsubscribe_all + punsubscribe_all to clean up.
