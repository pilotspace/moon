   Actually, the simpler and more correct approach: **use the same coordinator pattern for ALL multi-key cases** (whether cross-shard or not). This avoids code duplication and is correct.

   **Revised approach -- unified coordinator for multi-key:**

   a. For SINGLE key (keys.len() == 1): keep existing fast path (one oneshot, one registration, await it).

   b. For MULTI key (keys.len() > 1): use coordinator pattern:
      - Generate one `wait_id` from local registry
      - Create an `mpsc::channel(1)` as the coordination channel (first result wins)
      - For each key:
        - Create a `oneshot::channel()`
        - If key is local: register in local `BlockingRegistry` with this oneshot tx
        - If key is remote: send `BlockRegister` via SPSC to target shard with this oneshot tx
        - Spawn a lightweight forwarding task: `tokio::task::spawn_local(async move { if let Ok(Some(frame)) = rx.await { let _ = coord_tx.send(frame).await; } })`
      - Await the mpsc receiver (first result wins) with timeout/shutdown select
      - On wakeup: call `remove_wait(wait_id)` on local registry, send `BlockCancel { wait_id }` to all remote shards that were registered
      - On timeout/shutdown: same cleanup

   Wait -- `spawn_local` requires a LocalSet. Let me check if the shard runtime uses one.

   Actually, the shard's single-threaded runtime IS a LocalSet effectively (each shard runs on its own tokio current_thread runtime). But `spawn_local` still needs to be within a `LocalSet::run_until`. Let me check.

   **Simpler approach without spawn_local:** Use `tokio::select!` with a dynamic number of futures. Since we have a Vec of oneshot receivers, we can use `futures::future::select_all` or a manual loop. But `select_all` requires pinning.

   **Simplest correct approach:** Use an `mpsc::channel(1)` as aggregator. For each key's oneshot, we don't spawn -- instead we just make each `WaitEntry`'s `reply_tx` send into a shared coordination mechanism.

   **Actually even simpler:** Since `WaitEntry.reply_tx` is a `oneshot::Sender<Option<Frame>>`, and we need first-wins semantics, we can:
   - Create N oneshot pairs (one per key registration)
   - Use `tokio::select!` in a macro-free way: wrap all receivers into a FuturesUnordered and `.next().await`

   Here is the concrete implementation:

   ```rust
   use futures::stream::{FuturesUnordered, StreamExt};

   // Multi-key path (keys.len() > 1)
   let wait_id = blocking_registry.borrow_mut().next_wait_id();
   let mut receivers: FuturesUnordered<_> = FuturesUnordered::new();
   let mut registered_remote_shards: Vec<usize> = Vec::new();

   {
       let mut reg = blocking_registry.borrow_mut();
       let mut producers = dispatch_tx.borrow_mut();

       for key in &keys {
           let target = key_to_shard(key, num_shards);
           let (tx, rx) = tokio::sync::oneshot::channel::<Option<Frame>>();
           receivers.push(rx);

           if target == shard_id {
               // Local registration
               let entry = crate::blocking::WaitEntry {
                   wait_id,
                   cmd: blocked_cmd_factory(),
                   reply_tx: tx,
                   deadline,
               };
               reg.register(selected_db, key.clone(), entry);
           } else {
               // Remote registration via SPSC
               let msg = ShardMessage::BlockRegister {
                   db_index: selected_db,
                   key: key.clone(),
                   wait_id,
                   cmd: blocked_cmd_factory(),
                   reply_tx: tx,
               };
               let _ = producers[target].try_push(msg);
               if !registered_remote_shards.contains(&target) {
                   registered_remote_shards.push(target);
               }
           }
       }
   } // borrows dropped

   // Await first result
   let frame = if let Some(dl) = deadline {
       tokio::select! {
           result = receivers.next() => {
               match result {
                   Some(Ok(Some(frame))) => frame,
                   _ => Frame::Null,
               }
           }
           _ = tokio::time::sleep_until(dl) => Frame::Null,
           _ = shutdown.cancelled() => {
               Frame::Error(Bytes::from_static(b"ERR server shutting down"))
           }
       }
   } else {
       tokio::select! {
           result = receivers.next() => {
               match result {
                   Some(Ok(Some(frame))) => frame,
                   _ => Frame::Null,
               }
           }
           _ = shutdown.cancelled() => {
               Frame::Error(Bytes::from_static(b"ERR server shutting down"))
           }
       }
   };

   // Cleanup: cancel all remaining registrations
   blocking_registry.borrow_mut().remove_wait(wait_id);
   {
       let mut producers = dispatch_tx.borrow_mut();
       for shard_idx in &registered_remote_shards {
           let _ = producers[*shard_idx].try_push(ShardMessage::BlockCancel { wait_id });
       }
   }
   // Drop remaining receivers (senders on remote shards will get Err on send -- harmless)
   drop(receivers);
   frame
   ```
