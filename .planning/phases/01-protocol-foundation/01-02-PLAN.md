    2. Create `benches/resp_parsing.rs` with criterion benchmarks:

       ```rust
       use bytes::BytesMut;
       use criterion::{black_box, criterion_group, criterion_main, Criterion, BenchmarkId};
       use rust_redis::protocol::{Frame, ParseConfig, parse, serialize};
       ```

       Benchmark functions:

       a. `bench_parse_simple_string` -- parse `+OK\r\n` repeatedly. Create BytesMut, put bytes, call parse(), measure throughput.

       b. `bench_parse_bulk_string` -- parse `$5\r\nhello\r\n`. Measures length-prefixed string parsing.

       c. `bench_parse_integer` -- parse `:12345\r\n`. Measures integer decoding.

       d. `bench_parse_array` -- parse a 3-element array: `*3\r\n$3\r\nSET\r\n$3\r\nfoo\r\n$3\r\nbar\r\n`. This is the typical Redis command format.

       e. `bench_parse_inline` -- parse `SET foo bar\r\n`. Measures inline command performance.

       f. `bench_serialize_array` -- serialize a 3-element Array of BulkStrings. Measures encoding throughput.

       g. `bench_roundtrip` -- serialize then parse a complex frame. Measures full cycle.

       Each benchmark:
       - Uses `c.bench_function("name", |b| b.iter(|| { ... }))` pattern
       - Creates a fresh BytesMut for each iteration (or re-fills it) to avoid measuring empty-buffer fast path
       - Uses `black_box()` to prevent dead code elimination
       - Uses `ParseConfig::default()` for config

       Group into criterion_group and criterion_main:
       ```rust
       criterion_group!(benches,
           bench_parse_simple_string,
           bench_parse_bulk_string,
           bench_parse_integer,
           bench_parse_array,
           bench_parse_inline,
           bench_serialize_array,
           bench_roundtrip,
       );
       criterion_main!(benches);
       ```

    3. Run benchmarks once to verify they execute: `cargo bench -- --quick`
