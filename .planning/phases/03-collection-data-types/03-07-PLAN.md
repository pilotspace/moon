**Test cases to add:**

1. `test_hash_commands` -- HSET key field1 val1 field2 val2, verify HGET returns correct values, HGETALL returns all pairs, HDEL removes field, HLEN reflects changes.

2. `test_list_commands` -- LPUSH mylist a b c, verify LRANGE 0 -1 returns [c, b, a] (Pitfall 5 verification), RPUSH adds to end, LPOP/RPOP work, LLEN is correct.

3. `test_set_commands` -- SADD myset a b c, SMEMBERS returns all 3, SCARD is 3, SREM removes one, SISMEMBER checks membership, SINTER of two sets.

4. `test_sorted_set_commands` -- ZADD zs 1 a 2 b 3 c, ZRANGE zs 0 -1 returns [a, b, c] ordered by score, ZSCORE returns correct scores, ZRANK returns correct ranks, ZRANGEBYSCORE with -inf +inf returns all.

5. `test_type_command_all_types` -- Create string (SET), hash (HSET), list (LPUSH), set (SADD), sorted set (ZADD). Verify TYPE returns "string", "hash", "list", "set", "zset" respectively.

6. `test_wrongtype_error` -- SET key as string, then HSET on same key -> verify WRONGTYPE error. LPUSH on string key -> WRONGTYPE. GET on hash key -> WRONGTYPE.

7. `test_scan_basic` -- Create several keys, SCAN 0 to iterate, collect all results, verify all keys returned. Verify SCAN terminates (returns cursor 0).

8. `test_unlink` -- SET key, UNLINK key, verify GET returns nil. UNLINK multiple keys, verify count.

9. `test_auth_required` -- Start server with requirepass="testpass". Attempt GET before AUTH -> NOAUTH error. AUTH with wrong pass -> WRONGPASS. AUTH with correct pass -> OK. Then GET works.
   - Note: For this test, pass `requirepass: Some("testpass".to_string())` to ServerConfig.
   - May need a separate server instance since requirepass changes the config.
   - Use a non-multiplexed connection (as established in Phase 02-04) or a separate port.

10. `test_hscan` -- HSET key with several fields, HSCAN 0, collect results, verify all fields returned.

**Important patterns:**
- Each test should use a unique port to avoid conflicts (existing pattern).
- Use `tokio::time::sleep(Duration::from_millis(50))` after server spawn to ensure it's ready.
- For AUTH test, use raw redis connection or create server with requirepass config.
