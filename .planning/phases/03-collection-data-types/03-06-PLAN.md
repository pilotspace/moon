1. Create `src/server/expiration.rs`:
   - Add `pub async fn run_active_expiration(db: Arc<Mutex<Vec<Database>>>, shutdown: CancellationToken)`:
     - Create `tokio::time::interval(Duration::from_millis(100))`
     - Loop: select on interval.tick() vs shutdown.cancelled()
     - On tick: lock mutex, iterate all databases, call `expire_cycle(&mut db)` for each, release lock
     - On shutdown: log "Active expiration task shutting down" via tracing::info and break
   - Add `fn expire_cycle(db: &mut Database)`:
     - Record start time with `Instant::now()`, set budget = 1ms
     - Loop:
       - Call `db.keys_with_expiry()` to get keys with TTL
       - If empty, break
       - Sample min(20, keys.len()) random keys using `rand::seq::IndexedRandom::choose_multiple`
       - For each sampled key, if `db.is_key_expired(key)`, call `db.remove(key)`, count expired
       - If expired * 4 < sample_size OR elapsed >= budget, break (continue loop otherwise)
     - Per Pitfall 7: Lock, do ONE batch of work, release. Do NOT loop inside the lock. The loop here is acceptable because expire_cycle runs entirely within one lock acquisition, but the 1ms budget ensures it releases quickly.


1. In `src/config.rs`:
   - Add `requirepass` field to ServerConfig:
     ```rust
     #[arg(long)]
     pub requirepass: Option<String>,
     ```
   - Add a test: parse with --requirepass, verify field is set.

2. In `src/command/connection.rs`:
   - Add `pub fn auth(args: &[Frame], requirepass: &Option<String>) -> Frame`:
     - If args.len() != 1, return err_wrong_args("AUTH")
     - If requirepass is None, return `Frame::Error(Bytes::from_static(b"ERR Client sent AUTH, but no password is set"))`
     - Extract password from args[0], compare with requirepass. If match, return OK. If no match, return `Frame::Error(Bytes::from_static(b"WRONGPASS invalid username-password pair or user is disabled."))`
   - Note: AUTH handler returns Frame, but the connection loop needs to track `authenticated` state. The auth function should return a distinguishing response so the connection loop knows to set `authenticated = true` on OK.

3. In `src/server/connection.rs`:
   - Update `handle_connection` signature to accept `requirepass: Option<String>`:
     ```rust
     pub async fn handle_connection(
         stream: TcpStream,
         db: Arc<Mutex<Vec<Database>>>,
         shutdown: CancellationToken,
         requirepass: Option<String>,
     )
     ```
   - Add `let mut authenticated = requirepass.is_none();` (true if no password configured)
   - In the command processing, before dispatching, check auth:
     - Extract command name (uppercase) from frame before dispatch
     - If `!authenticated`:
       - If command is "AUTH": call `connection::auth(cmd_args, &requirepass)`. If result is OK, set `authenticated = true`. Send result.
       - If command is anything else: send `Frame::Error(Bytes::from_static(b"NOAUTH Authentication required."))` and continue loop.
     - If `authenticated`: dispatch normally.
     - IMPORTANT: The dispatch function currently extracts the command name. For the auth gate, we need to peek at the command name before dispatching. Extract the first Frame from the array, uppercase it, check if it's AUTH. If so, handle directly. Otherwise, if not authenticated, reject.
   - This requires restructuring the handle_connection loop slightly: parse the frame into args BEFORE calling dispatch, check auth, then either handle AUTH directly or call dispatch.

4. In `src/server/listener.rs`:
   - Update the `handle_connection` call to pass `config.requirepass.clone()`:
     ```rust
     let requirepass = config.requirepass.clone();
     tokio::spawn(connection::handle_connection(stream, db, conn_token, requirepass));
     ```

5. In `src/command/mod.rs`:
   - Add `b"AUTH"` to dispatch (routing to connection::auth), OR handle AUTH entirely in connection.rs handle_connection (since it needs the requirepass and mutable authenticated state). The cleaner approach is to handle AUTH in the connection loop directly, not through dispatch. Keep dispatch unaware of AUTH.

6. Add unit test for auth function: correct password returns OK, wrong password returns WRONGPASS error, no password configured returns ERR.
