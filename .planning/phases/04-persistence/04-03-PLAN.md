  <action>
Add persistence integration tests to `tests/integration.rs`. Use `tempdir` (or `std::env::temp_dir` + unique subdir) for persistence files to avoid test interference.

Each test:
1. Create ServerConfig with persistence flags and a unique temp directory
2. Spawn server with `run_with_shutdown` and a CancellationToken
3. Connect with redis crate client
4. Execute commands
5. Cancel server, verify files/state

Tests to add:

**test_bgsave_creates_rdb_file:**
- Start server with `--dir /tmp/test_X --dbfilename dump.rdb`
- SET key1 value1, SET key2 value2, HSET myhash f1 v1
- Send BGSAVE command
- Wait briefly (100ms) for background save to complete
- Verify dump.rdb file exists in temp dir
- Verify file starts with RUSTREDIS magic bytes

**test_rdb_restore_on_startup:**
- Start server 1, SET several keys of different types, BGSAVE, stop server 1
- Start server 2 with same --dir and --dbfilename
- GET the keys, verify they have correct values
- Verify all data types restored (string, hash, list, set, sorted set)

**test_aof_logging:**
- Start server with `--appendonly yes --dir /tmp/test_X`
- SET foo bar, HSET hash f v, LPUSH list a
- Stop server (graceful shutdown flushes AOF)
- Read appendonly.aof file, verify it contains RESP-formatted SET/HSET/LPUSH commands

**test_aof_restore_on_startup:**
- Start server 1 with AOF enabled, SET keys, stop
- Start server 2 with same AOF config and dir
- GET the keys, verify values match

**test_aof_priority_over_rdb:**
- Start server with both RDB and AOF
- SET key1 "rdb_value", BGSAVE
- SET key1 "aof_value" (overwrites in AOF but not in RDB)
- Stop server
- Restart with AOF enabled -- GET key1 should return "aof_value"

**test_bgrewriteaof:**
- Start server with AOF, SET key1 many times (creates large AOF)
- Send BGREWRITEAOF
- Wait briefly for rewrite
- Verify AOF file size decreased or stayed same
- Stop and restart server, verify key1 has final value

Use the existing test helper pattern from `tests/integration.rs` for spawning servers on unique ports.
