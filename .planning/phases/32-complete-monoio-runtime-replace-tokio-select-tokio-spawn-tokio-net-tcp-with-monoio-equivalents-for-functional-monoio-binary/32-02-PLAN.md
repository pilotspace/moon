<objective>
Implement the Monoio connection handler that reads/writes RESP frames using monoio's ownership-based I/O model, wiring it into the shard event loop.

Purpose: This is the hardest part of the migration. Monoio's `AsyncReadRent` takes buffer ownership (`stream.read(buf)` returns `(result, buf)`) unlike Tokio's borrow-based `stream.read_buf(&mut buf)`. The connection handler must manage buffer ownership while parsing RESP frames and dispatching commands.

Output: Working connection handler that can process SET/GET/PING through the existing dispatch infrastructure.
</objective>

Create a `#[cfg(feature = "runtime-monoio")]` version of the sharded connection handler. The key challenge is translating the tokio readability-wait + try_read pattern to monoio's ownership-based I/O.

**Strategy:** Use monoio's `AsyncReadRent::read()` which takes buffer ownership. Since `BytesMut` implements `monoio::buf::IoBufMut`, we can pass it directly. However, `BytesMut` may NOT implement monoio's buf traits. If not, use `Vec<u8>` as the read buffer and copy into `BytesMut` for codec parsing.

Add the following function to connection.rs:

```rust
#[cfg(feature = "runtime-monoio")]
pub async fn handle_connection_sharded_monoio(
    stream: monoio::net::TcpStream,
    databases: Rc<RefCell<Vec<Database>>>,
    shard_id: usize,
    num_shards: usize,
    dispatch_tx: Rc<RefCell<Vec<HeapProd<ShardMessage>>>>,
    pubsub_registry: Rc<RefCell<PubSubRegistry>>,
    blocking_registry: Rc<RefCell<crate::blocking::BlockingRegistry>>,
    shutdown: CancellationToken,
    aof_tx: Option<channel::MpscSender<AofMessage>>,
    tracking_table: Rc<RefCell<TrackingTable>>,
    client_id: u64,
    repl_state: Option<Arc<RwLock<crate::replication::state::ReplicationState>>>,
    cluster_state: Option<Arc<RwLock<crate::cluster::ClusterState>>>,
    lua: std::rc::Rc<mlua::Lua>,
    script_cache: std::rc::Rc<std::cell::RefCell<crate::scripting::ScriptCache>>,
    config_port: u16,
    acl_table: Arc<RwLock<crate::acl::AclTable>>,
    runtime_config: Arc<RwLock<RuntimeConfig>>,
    spsc_notifiers: Vec<std::sync::Arc<channel::Notify>>,
) {
    use monoio::io::{AsyncReadRent, AsyncWriteRent};

    let peer_addr = stream.peer_addr()
        .map(|a| a.to_string())
        .unwrap_or_else(|_| "unknown".to_string());

    let mut read_buf = BytesMut::with_capacity(8192);
    let mut write_buf = BytesMut::with_capacity(8192);
    let mut codec = crate::server::codec::RespCodec::default();
    let mut selected_db: usize = 0;
    let mut authenticated = true; // No requirepass in MVP monoio path
    let mut current_user = "default".to_string();
    let mut protocol_version: u8 = 2;

    // Per-connection arena
    let mut arena = bumpalo::Bump::with_capacity(4096);

    // Transaction state
    let mut in_multi = false;
    let mut command_queue: Vec<Frame> = Vec::new();

    // Tracking state
    let mut tracking_state = TrackingState::default();
    let mut tracking_rx: Option<channel::MpscReceiver<Frame>> = None;
    let mut client_name: Option<Bytes> = None;
    let mut asking = false;
    let mut acl_log = crate::acl::AclLog::new(128);

    loop {
        // Read data from stream using ownership I/O
        // Allocate a temporary read buffer for monoio
        let tmp_buf = vec![0u8; 8192];
        let (result, tmp_buf) = stream.read(tmp_buf).await;
        match result {
            Ok(0) => break, // connection closed
            Ok(n) => {
                read_buf.extend_from_slice(&tmp_buf[..n]);
            }
            Err(_) => break,
        }

        // Parse all complete frames
        let mut frames = Vec::new();
        loop {
            match codec.decode_frame(&mut read_buf) {
                Ok(Some(frame)) => frames.push(frame),
                Ok(None) => break,
                Err(_) => { /* parse error, close connection */ return; }
            }
        }

        if frames.is_empty() {
            continue;
        }

        // Process frames -- reuse the SAME dispatch logic as tokio path
        // For MVP: process each frame individually through dispatch
        write_buf.clear();
        for frame in frames {
            // Extract command and dispatch
            if let Some((cmd, cmd_args)) = extract_command(&frame) {
                // Handle QUIT
                if cmd.eq_ignore_ascii_case(b"QUIT") {
                    let quit_resp = Frame::SimpleString(Bytes::from_static(b"OK"));
                    codec.encode_frame(&quit_resp, &mut write_buf);
                    let data = write_buf.split().freeze();
                    let (res, _) = stream.write_all(data).await;
                    let _ = res;
                    return;
                }

                // Dispatch to database
                let mut dbs = databases.borrow_mut();
                let db = &mut dbs[selected_db];
                let response = crate::command::dispatch(
                    cmd, cmd_args, db,
                    &mut in_multi, &mut command_queue,
                    &aof_tx, None, // change_counter
                );
                let resp = match response {
                    crate::command::DispatchResult::Response(f) => f,
                    crate::command::DispatchResult::Quit => {
                        let quit_resp = Frame::SimpleString(Bytes::from_static(b"OK"));
                        codec.encode_frame(&quit_resp, &mut write_buf);
                        let data = write_buf.split().freeze();
                        let (res, _) = stream.write_all(data).await;
                        let _ = res;
                        return;
                    }
                };
                drop(dbs);
                codec.encode_frame(&resp, &mut write_buf);
            } else {
                // Invalid command format
                let err = Frame::Error(Bytes::from_static(b"ERR invalid command format"));
                codec.encode_frame(&err, &mut write_buf);
            }
        }

        // Write all responses in one batch
        if !write_buf.is_empty() {
            let data = write_buf.split().freeze();
            let (result, _) = stream.write_all(data).await;
            if result.is_err() {
                break;
            }
        }

        // Check shutdown
        if shutdown.is_cancelled() {
            break;
        }

        // Shrink buffers if they grew too large
        if read_buf.capacity() > 65536 {
            read_buf = BytesMut::with_capacity(8192);
        }
    }
}
```

**IMPORTANT NOTES for the executor:**

1. Check if `BytesMut` implements monoio's `IoBufMut` trait. If yes, use `stream.read(read_buf)` directly instead of the `vec![0u8; 8192]` intermediate. If not, the vec intermediate is fine for MVP.

2. For `stream.write_all()`, monoio takes ownership of the buffer. Use `write_buf.split().freeze()` to get a `Bytes` which implements `IoBuf`. If `Bytes` doesn't impl `IoBuf`, convert to `Vec<u8>` via `write_buf.to_vec()`.

3. The dispatch path reuses the existing `crate::command::dispatch()` function which is runtime-agnostic. This is the same function the tokio path uses.

4. Skip these features for MVP monoio handler (they're tokio-only for now):
   - Pub/sub subscriber mode
   - Blocking commands (BLPOP etc.)
   - Client tracking / invalidation
   - MULTI/EXEC across shards
   - Cluster routing (MOVED/ASK)
   - Replication (PSYNC)
   - ACL enforcement (allow all commands)
   Keep the parameters in the signature for future use but don't wire them.

5. The `select!` shutdown check is replaced with a simple `is_cancelled()` poll after each batch, which is acceptable for MVP (adds at most one read-wait latency to shutdown).

6. Ensure all necessary imports are cfg-gated. The function needs `#[cfg(feature = "runtime-monoio")]` and should import `monoio::io::{AsyncReadRent, AsyncWriteRent}` inside the function body.
