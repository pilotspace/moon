<objective>
Cache the current timestamp in a per-shard AtomicU64 updated once per 1ms event loop tick, eliminating 4% CPU overhead (66 samples) from clock_gettime syscalls in the 8-shard p=16 profiling workload.

Purpose: `Database::refresh_now()` calls `SystemTime::now()` on every batch/message, generating 66 samples of clock_gettime overhead. Redis caches time once per event loop iteration. We do the same: update an AtomicU64 once per 1ms tick, read it everywhere else.

Output: Shared timestamp cache readable by both shard event loop and connection handlers.
</objective>

1. In `src/storage/entry.rs`, add a shared timestamp cache type:

```rust
use std::sync::Arc;
use std::sync::atomic::{AtomicU64, Ordering};

/// Shared cached clock updated once per shard event loop tick.
/// Packs seconds (upper 32 bits) and millis (lower 32 bits of the ms value)
/// into a single AtomicU64 to avoid two atomic loads.
/// Actually simpler: just use two AtomicU64s -- one for secs, one for ms.
#[derive(Clone)]
pub struct CachedClock {
    secs: Arc<AtomicU64>,
    ms: Arc<AtomicU64>,
}

impl CachedClock {
    pub fn new() -> Self {
        let now_secs = current_secs() as u64;
        let now_ms = current_time_ms();
        Self {
            secs: Arc::new(AtomicU64::new(now_secs)),
            ms: Arc::new(AtomicU64::new(now_ms)),
        }
    }

    /// Update the cached clock. Called once per shard tick (1ms).
    #[inline]
    pub fn update(&self) {
        self.secs.store(current_secs() as u64, Ordering::Relaxed);
        self.ms.store(current_time_ms(), Ordering::Relaxed);
    }

    /// Read cached seconds.
    #[inline]
    pub fn secs(&self) -> u32 {
        self.secs.load(Ordering::Relaxed) as u32
    }

    /// Read cached milliseconds.
    #[inline]
    pub fn ms(&self) -> u64 {
        self.ms.load(Ordering::Relaxed)
    }
}

impl Default for CachedClock {
    fn default() -> Self { Self::new() }
}
```

Relaxed ordering is correct here: timestamp monotonicity is not required for correctness (expiry checks tolerate stale values up to 1ms), and we avoid the cost of acquire/release fences.

fn test_cached_clock_initial_values() {
    let clock = CachedClock::new();

#[test]
fn test_cached_clock_update() {
    let clock = CachedClock::new();
    let before = clock.ms();
    std::thread::sleep(std::time::Duration::from_millis(5));
    clock.update();
    let after = clock.ms();
    assert!(after >= before);
}

#[test]
fn test_cached_clock_clone_shares_state() {
    let clock1 = CachedClock::new();

    assert_eq!(clock1.ms(), clock2.ms());

}
