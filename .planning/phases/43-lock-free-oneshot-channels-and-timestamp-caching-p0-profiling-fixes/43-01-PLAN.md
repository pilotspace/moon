<objective>
Replace the flume-based oneshot channel with a lock-free implementation using AtomicU8 state machine + UnsafeCell + AtomicWaker. This eliminates 12% CPU overhead (216 samples) from pthread_mutex contention in the 8-shard p=16 profiling workload.

Purpose: The current `channel::oneshot()` wraps `flume::bounded(1)` which uses `Arc<Shared<Mutex<VecDeque>>>`. Every cross-shard command creates one, and both send and recv contend on the mutex between two threads. The lock-free version uses atomic operations only.

Output: Drop-in replacement oneshot in `src/runtime/channel.rs` with identical public API.
</objective>

2. In `src/runtime/channel.rs`, replace the flume-based oneshot with a lock-free implementation:

```rust
use std::cell::UnsafeCell;
use std::mem::MaybeUninit;
use std::sync::atomic::{AtomicU8, Ordering};
use std::sync::Arc;
use atomic_waker::AtomicWaker;
```

Define state constants:
```rust
const EMPTY: u8 = 0;
const VALUE: u8 = 1;
const CLOSED: u8 = 2;
```

Define inner struct:
```rust
struct OneshotInner<T> {
    state: AtomicU8,
    value: UnsafeCell<MaybeUninit<T>>,
    waker: AtomicWaker,
}
unsafe impl<T: Send> Send for OneshotInner<T> {}
unsafe impl<T: Send> Sync for OneshotInner<T> {}
```

Implement `OneshotSender<T>`:
- `send(self, value: T) -> Result<(), T>`: Write value to UnsafeCell, compare_exchange state EMPTY->VALUE. If state was already CLOSED, read back value and return Err. Call `waker.wake()`.
- `Drop`: If state is EMPTY, set to CLOSED and wake (receiver gets error).

Implement `OneshotReceiver<T>`:
- `recv(self) -> Result<T, RecvError>`: Await self (uses Future impl).
- `Future` impl: Load state. If VALUE, read value from UnsafeCell, return Poll::Ready(Ok(val)). If CLOSED, return Poll::Ready(Err(RecvError)). If EMPTY, register waker via `inner.waker.register(cx.waker())`, re-check state (avoids race), return Pending.
- `Drop`: If state is EMPTY, set to CLOSED (so sender's send returns Err).

Define `RecvError` as a simple unit struct (replacing `flume::RecvError`). Update the public API:
```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub struct RecvError;
impl std::fmt::Display for RecvError { ... }
impl std::error::Error for RecvError {}
```

The `oneshot()` function signature stays identical:
```rust
pub fn oneshot<T>() -> (OneshotSender<T>, OneshotReceiver<T>)
```

3. Update ALL callers that reference `flume::RecvError` to use the new `RecvError` type. Grep for `flume::RecvError` -- it appears in:

4. Keep the existing `mpsc`, `watch`, `Notify` implementations using flume (only oneshot changes).

5. Update tests: replace `rx.0.recv()` (direct flume access) with proper test patterns. Add the new tests from the behavior block above. For async tests without tokio, use `futures::executor::block_on()`.

IMPORTANT: The `OneshotReceiver` Future impl must handle the race where send happens between state check and waker registration. After registering the waker, re-load state and check again before returning Pending.

IMPORTANT: Memory ordering -- use Acquire on loads, Release on stores for state transitions. The UnsafeCell write happens-before the Release store of VALUE, and the Acquire load of VALUE happens-before the UnsafeCell read.
