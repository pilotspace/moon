<objective>
Implement inline dispatch shortcut for single-shard GET/SET in the Monoio handler. When num_shards == 1, all commands are local — for GET and SET we can skip Frame construction, extract_command, and dispatch table lookup entirely. Instead: parse command type + key directly from raw read_buf bytes, do DashTable lookup/insert, serialize response directly into write_buf.

Purpose: Eliminate Frame allocation overhead for the two most common commands in single-shard mode, targeting >1.4x Redis at p=1.
Output: Modified Monoio handler with inline dispatch fast path.
</objective>

  <action>
Create a `try_inline_dispatch` function near the Monoio handler (around line 3100) that takes `read_buf: &mut BytesMut`, `write_buf: &mut BytesMut`, `databases: &Rc<RefCell<Vec<Database>>>`, `selected_db: usize`, `aof_tx: &Option<channel::MpscSender<AofMessage>>` and returns `usize` (number of commands processed, 0 if none could be inlined).

The function should:

1. **Detect GET pattern** in read_buf: `*2\r\n$3\r\nGET\r\n$` prefix (case-insensitive on "GET").
   - Use memchr to find \r\n boundaries for the key length field
   - Parse the key length (use atoi or manual digit parsing)
   - Verify full command is present in buffer (including trailing \r\n)
   - Extract key bytes as a zero-copy slice from read_buf
   - Call `databases.borrow_mut()[selected_db].get(key_bytes)`
   - If found and value `as_bytes()` returns Some(val):
     - Write `$` + itoa(val.len()) + `\r\n` + val + `\r\n` directly to write_buf
   - If found but not a string type: write `-WRONGTYPE Operation against a key holding the wrong kind of value\r\n`
   - If not found: write `$-1\r\n` (null bulk string)
   - Advance read_buf past the consumed command bytes
   - Return 1

2. **Detect SET pattern** in read_buf: `*3\r\n$3\r\nSET\r\n$` prefix (case-insensitive on "SET").
   - Only handle the simple 3-argument SET (no EX/PX/NX/XX options — those have *5, *7 etc)
   - Parse key length, key bytes, value length, value bytes from read_buf
   - Verify full command present
   - Create Entry with CompactValue from value bytes, call `databases.borrow_mut()[selected_db].set(key, entry)`
   - If aof_tx is Some, serialize the SET command to AOF: `aof::serialize_command_raw(b"SET", &[key_bytes, val_bytes])` or reconstruct the RESP bytes from read_buf slice before advancing
   - Write `+OK\r\n` to write_buf
   - Advance read_buf past the consumed command bytes
   - Return 1

3. **Loop**: Call try_inline_dispatch in a loop to process multiple consecutive GET/SET commands from the same read_buf. Stop when it returns 0 (non-inlineable command found, or buffer incomplete).

4. **Integration point**: In the main Monoio handler loop, BEFORE the existing frame parsing loop (line ~3383), add:
   ```rust
   if num_shards == 1 {
       // Inline dispatch: handle GET/SET without Frame construction
       let inlined = try_inline_dispatch_loop(&mut read_buf, &mut write_buf, &databases, selected_db, &aof_tx);
       if inlined > 0 && read_buf.is_empty() {
           // All commands were inlined — flush write_buf and continue
           if !write_buf.is_empty() {
               let data = write_buf.split().freeze();
               let (result, _): (std::io::Result<usize>, bytes::Bytes) = stream.write_all(data).await;
               if result.is_err() { break; }
           }
           continue;
       }
       // If read_buf still has data, fall through to normal Frame parsing for remaining commands
       // The inlined responses are already in write_buf and will be flushed with the rest
   }
   ```

5. **Important edge cases**:
   - Case-insensitive command matching (GET/get/Get all valid)
   - If read_buf contains a partial command, return 0 (let normal path handle it)
   - The function must NOT consume bytes it can't fully process
   - For SET, use `Bytes::copy_from_slice` for the key and value to create owned copies before advancing read_buf
   - Refresh `cached_now_ms` once before the inline dispatch loop (same as the batch refresh at line 3412)

6. **Do NOT** inline any command beyond GET (3-arg) and SET (3-arg). MULTI, SUBSCRIBE, AUTH, blocking commands, and even GET/SET with options all fall through to the normal path.
