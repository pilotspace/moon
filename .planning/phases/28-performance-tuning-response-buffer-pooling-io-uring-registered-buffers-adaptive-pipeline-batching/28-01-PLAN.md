Add a registered send buffer pool to the io_uring driver and wire it into the shard event loop for zero-kernel-copy writes.

Purpose: Eliminate per-response BytesMut allocations on the io_uring send path and remove per-I/O get_user_pages() kernel overhead by pre-registering send buffers with IORING_REGISTER_BUFFERS + IORING_OP_WRITE_FIXED.

Output: UringDriver manages a fixed pool of pre-registered send buffers; shard event loop serializes responses into pooled buffers and reclaims them on SendComplete.
</objective>

  <action>
Add a send buffer pool to UringDriver with pre-registered kernel buffers:

1. Add a `SendBufPool` struct (inside uring_driver.rs, not a separate file):
   - `buffers: Vec<Vec<u8>>` — the actual buffer storage (each 8KB, enough for most responses)
   - `free_list: Vec<u16>` — indices of available buffers (stack-based LIFO for cache locality)
   - `buf_count: u16` — total buffer count (default 256 per shard = 2MB)
   - `buf_size: usize` — per-buffer size (8192 bytes default)

2. In `UringDriver::init()`, after FD table and buf_ring registration:
   - Create `buf_count` buffers via `Vec::with_capacity(buf_size)` and set_len to buf_size
   - Build an `iovec` array pointing to each buffer
   - Call `io_uring::Submitter::register_buffers()` with the iovec array
   - Populate free_list with all indices

3. Add `SendBufPool` methods:
   - `alloc(&mut self) -> Option<(u16, &mut [u8])>` — pop from free_list, return (index, slice)
   - `reclaim(&mut self, idx: u16)` — push index back to free_list
   - `is_exhausted(&self) -> bool` — free_list is empty

4. Add `UringDriver` methods:
   - `submit_send_fixed(&mut self, conn_id: u32, buf_idx: u16, len: u32) -> io::Result<()>` — uses `opcode::WriteFixed` with `types::Fixed(conn.fixed_fd_idx)`, buf_idx encoded in user_data aux field
   - `alloc_send_buf(&mut self) -> Option<(u16, &mut [u8])>` — delegates to SendBufPool::alloc
   - `reclaim_send_buf(&mut self, buf_idx: u16)` — delegates to SendBufPool::reclaim

5. For `submit_send_fixed`, use `opcode::Write::new(types::Fixed(fd), ptr, len).buf_group(buf_idx)` with the `.flags(Flags::FIXED_FILE)` already implied by `types::Fixed`. Encode buf_idx in the aux field of user_data via `encode_user_data(EVENT_SEND, conn_id, buf_idx as u32)`.

6. Add `send_buf_pool_size` to `UringConfig` (default 256).

7. Keep existing `submit_send()` and `submit_writev()` as fallbacks for when the pool is exhausted.

Do NOT modify the recv-side provided buffer ring — that path is already optimized.
Do NOT change `WritevGuard` or `submit_writev_bulkstring` — scatter-gather GET still uses writev.

Update `handle_uring_event` and related shard code to use pooled send buffers:

1. Replace `InFlightSend::Buf(BytesMut)` with `InFlightSend::Fixed(u16)` variant that tracks the pooled buffer index. Keep `InFlightSend::Writev(WritevGuard)` unchanged for scatter-gather GET path.

2. In `handle_uring_event` IoEvent::Recv handler, for the non-BulkString response path (lines ~1045-1057):
   - Try `driver.alloc_send_buf()` first
   - If Some((buf_idx, buf_slice)): serialize response into buf_slice, call `driver.submit_send_fixed(conn_id, buf_idx, len)`, push `InFlightSend::Fixed(buf_idx)` to inflight_sends
   - If None (pool exhausted): fall back to existing BytesMut allocation + submit_send path (keep `InFlightSend::Buf` variant for this)

3. In `handle_uring_event` IoEvent::SendComplete handler (lines ~1089+):
   - When popping from inflight_sends, match on variant:
     - `InFlightSend::Fixed(idx)` => call `driver.reclaim_send_buf(idx)`
     - `InFlightSend::Writev(_)` => drop (existing behavior)
     - `InFlightSend::Buf(_)` => drop (existing fallback behavior)

4. Extract buf_idx from user_data aux field in SendComplete: decode with `decode_user_data` which already returns `(event_type, conn_id, aux)`. The aux field carries the buf_idx when InFlightSend::Fixed was used.

5. In IoEvent::Disconnect handler, reclaim ALL pooled buffers for the disconnecting connection before removing from inflight_sends.

6. All existing tests must continue passing — the fallback path ensures correctness even if pool is exhausted.
