**mesh.rs:** Replace:
- `tokio::sync::mpsc::Sender<ShardMessage>` -> `crate::runtime::channel::MpscSender<ShardMessage>`
- `tokio::sync::mpsc::Receiver<ShardMessage>` -> `crate::runtime::channel::MpscReceiver<ShardMessage>`
- `Arc<tokio::sync::Notify>` -> `Arc<crate::runtime::channel::Notify>`

- `tokio::sync::Notify::new()` -> `crate::runtime::channel::Notify::new()`
- The `mpsc::Sender` for connection distribution (conn_tx/conn_rx) also uses tokio mpsc -- replace with flume bounded channel.
- Update `recv()` calls: flume uses `.recv_async().await` instead of `.recv().await`.

**coordinator.rs:** Replace all `tokio::sync::oneshot::channel()` with `crate::runtime::channel::oneshot()` and `tokio::sync::oneshot::Receiver<T>` with `crate::runtime::channel::OneshotReceiver<T>`. Update `.await` patterns -- flume oneshot receiver uses `.recv()` (which is already async via our wrapper).

**mod.rs (shard event loop):** This is the trickiest file:
- Replace `tokio::sync::mpsc` import with `crate::runtime::channel`
- Replace `tokio_util::sync::CancellationToken` with `crate::runtime::cancel::CancellationToken`
- Replace `tokio::sync::watch::Receiver<u64>` with `crate::runtime::channel::WatchReceiver<u64>`
- Replace `tokio::sync::oneshot::Sender/channel` with `crate::runtime::channel::OneshotSender/oneshot`
- Replace `Arc<tokio::sync::Notify>` with `Arc<crate::runtime::channel::Notify>`
- **For `tokio::select!`**: Replace with `futures::select!` from the futures crate (already a dependency). Key differences:
  - `futures::select!` requires `FusedFuture` or use `futures::select_biased!`
  - Simpler alternative: use `#[cfg(feature = "runtime-tokio")]` block with `tokio::select!` and `#[cfg(feature = "runtime-monoio")]` block with equivalent logic using `futures::select!`
  - The shard event loop has 6 arms in its select -- cfg-gate the entire select block
- Replace `.recv().await` on mpsc receivers with `.recv_async().await` (flume API)
- Replace `.changed().await` on watch receiver with runtime::channel watch API
- Replace `.notified().await` on Notify with runtime::channel Notify API

**Important**: For `tokio::select!`, use cfg-gated blocks:
```rust
#[cfg(feature = "runtime-tokio")]
{
    tokio::select! { ... }
}
#[cfg(feature = "runtime-monoio")]
{
    // Use futures::select! or sequential poll approach
    futures::select! { ... }
}
```
Or better: define a macro in `src/runtime/mod.rs` that wraps the appropriate select.
