  Pipelined 4-shard (the A2 gate): SET +368%, GET +19% — NO regression.
  4-shard non-pipelined SET +404% (29K -> 148K): cross-shard writes no longer
  wait for the tick. GET s4 P1/c1 within run-to-run VM noise (-5..-7%) —
  expected: cross-shard READS take the shared-read fastpath (no SPSC message),
  so the wake change does not touch their path.
  Single-shard improvements (+63..+80%) come from the same loop no longer
  oversleeping its own local work between ticks.
